## 一、配置

### 1. 会计科目表

#### 字段解释

| 字段 | 说明 |
|------|------|
| **科目类型** | 对应资产负债表大类 |
| **允许对账** | 应收应付类型的才会勾选，对应的是会计轧账 |
| **币别** | 选择了只有这个币别才能进入这个科目 |
| **有效** | 废弃 |
| **公司** | 科目所属公司 |
| **集团化** | 公用同一套会计科目表 |
| **标签** | Odoo 原生功能，用于现金流量表的标签 |
| **配对** | 集团公司的科目映射 |

#### 如何设置二级科目

- **应收应付类型科目**：会自动关联【合作伙伴】，反馈在业务伙伴分类账及应收应付报表中
- **其他科目**：若需要细化到子科目，没有严格的控制策略，根据前缀相同即可。其目的是为了财务报表取数（取科目的前 4 位），例如：费用、银行

#### 科目组

科目组用于固化科目之间的关系，体现在资产负债表中：打开层级和小计，增加一层按照科目组进行汇总。

### 2. 日记账配置（会计账簿）、银行日记账配置方法

> **注意**：Odoo 做的是明细凭证，不是总账凭证。

**日记账**：每个银行账户建立一个账本。

#### 付款为例

**情况一：产生中间凭证**

- 银行流水产生时：`借：暂记 / 贷：银行`
- 核销时：`借：应付 / 贷：暂记`

**情况二：不产生中间凭证**

- 核销时：`借：应付 / 贷：银行`

### 6. 科目映射、税率映射

应收应付与合作伙伴的科目映射：会计 → 配置 → 应税设定，维护在合作伙伴上。

税率映射/财务状况：用于产品上默认含税，特定的伙伴需要调整税额的情况。

**配置步骤**：

1. 建立应税设定
2. 在税率上选择应税设定，而后选择替换税率

![](https://cdn.nlark.com/yuque/0/2026/png/38859133/1772267380766-87a9807a-b596-45d8-82f5-10f290900552.png)

3. 在供应商上绑定应税设定

### 10. 期末结转规则配置

**菜单**：会计 → 转账

- 配置自动结转规则，每个月 1 日结转上个月的凭证
- 损益类结转建立一个单独的日记账【月末结转】、杂项，结转的凭证到该日记账下
- 也可以不结转，报表取数取到即可

> **待补充**：具体科目设置示例（如"本年利润"结转规则）、结转凭证样例

---

### 11. 关账日期设置

**菜单**：会计 → 会计 → 锁定日期

- 只锁定财务凭证，不锁定单据
- **反关账操作**：例如确实需要补账的，关账时间选择今天之后

> **注意**：反关账会留下审计痕迹，需谨慎操作。
