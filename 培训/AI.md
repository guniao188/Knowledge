## 什么是 Agent

**Agent** 可以理解成"会自己分步骤完成任务的 AI 助手"。

普通 AI 更像是"你问一句，它答一句"；Agent 更像是"你给它一个目标，它会自己规划并推进任务"。它可以拆解步骤、查资料、调用工具、写文件、运行代码、检查结果，并根据结果继续下一步。

比如：

- 普通 AI：你问"帮我写一份 ERP 培训大纲"，它直接生成文字。
- Agent：你说"帮我做一份 ERP 培训 PPT"，它可能会先列大纲、生成内容、查找或生成图片、制作 PPT、检查排版，最后输出文件。

### Agent 的核心能力

| 能力 | 说明 |
|------|------|
| **任务规划** | 将复杂目标拆解为可执行的步骤 |
| **工具调用** | 自主调用搜索、代码执行、文件操作等工具 |
| **上下文理解** | 理解整个代码库或项目结构，而非仅当前文件 |
| **迭代执行** | 根据执行反馈不断调整，直到任务完成 |
| **多代理协作** | 多个 Agent 分工协作，同时处理不同子任务 |

## 什么是 Skill

**Skill** 可以理解成"AI 的专项技能包"。

AI 本身什么都能聊一点，但 Skill 会给 AI 加上某个领域或工具的专门能力，让它在特定任务上更专业、更稳定。

比如：

- **文档 Skill**：专门处理 Word、培训材料、合同草稿
- **表格 Skill**：专门分析 Excel、生成公式、做数据统计
- **客服 Skill**：按照公司话术和知识库回复客户问题
- **代码审查 Skill**：自动检查代码规范、安全漏洞、性能问题
- **部署 Skill**：自动化打包、发布、回滚等运维操作

Skill 可以被团队共享和复用。例如，你可以创建一个 `/review-pr` Skill，让 AI 按照团队标准审查 Pull Request；或者创建一个 `/deploy-staging` Skill，一键部署到测试环境。

> **Skill 下载地址**：https://agentskill.sh/；https://www.skills.sh/

## 什么是 MCP

**MCP（Model Context Protocol，模型上下文协议）** 是连接 AI 与外部工具的开放标准。

可以把 MCP 理解为 AI 的"万能转接头"——它让 AI 能够安全地访问你的各种数据源和工具，比如：

| 类型 | 示例 |
|------|------|
| 文档存储 | Google Drive、Notion、Confluence |
| 项目管理 | Jira、Linear、Trello |
| 通讯工具 | Slack、Discord、企业微信 |
| 数据库 | PostgreSQL、MySQL、MongoDB |
| 开发工具 | GitHub、GitLab、Figma |
| 自定义工具 | 公司内部 API、私有数据源 |

通过 MCP，AI Agent 不再局限于"聊天窗口"，而是真正融入你的工作流——自动读取设计文档、更新任务状态、拉取业务数据、操作内部系统等。

## 可选的工作模式

| 模式 | 说明 | 适合场景 |
|------|------|----------|
| **请求批准** | 编辑外部文件、访问互联网等操作都会先询问 | 对安全性要求高，或正在处理重要文件 |
| **替我审批** | 仅在检测到风险操作时请求确认 | 日常办公、开发和文档处理 |
| **完全访问权限** | 可不受限制地访问互联网和本机文件 | 受信任环境下的高效率自动化任务 |

> 💡 **建议**：初次使用或处理敏感数据时选择"请求批准"模式；熟悉后日常开发可选用"替我审批"模式以提升效率。

## 强大的计划模式

先让 AI 列出步骤/方案，然后逐一确认后再执行。

计划模式特别适用于：
- 复杂功能的开发（涉及多个文件修改）
- 重构或架构调整
- 需要多步骤配合的自动化任务
- 对结果有明确预期，希望把控过程的场景
