## 一、仓库基础配置

### 1. 仓库与库位

#### 字段解释

| 字段 | 说明 |
|------|------|
| **仓库** | 独立的库存管理单位，有自己的收货/发货/库存操作 |
| **库位 (Location)** | 仓库内的具体位置，可多层嵌套 |
| **库位类型** | 供应商库位 / 视图 / 内部库位 / 客户库位 / 盘点库位 / 废品库位 / 退货库位 / 生产库位 |
| **父库位** | 上级位置，形成层级树 |
| **存放规则** | 产品存放的约束条件 |
| **容量** | 库位的最大存放容量 |

#### 库位层级示例

```
WH/Stock (仓库/库存区)
├── WH/Stock/Shelf-A (货架A)
│   ├── WH/Stock/Shelf-A/Level-1
│   └── WH/Stock/Shelf-A/Level-2
├── WH/Stock/Shelf-B
└── WH/Stock/Cold-Room (冷库)
```

#### 配置步骤

1. **创建仓库**
   - 库存 → 配置 → 仓库 → 新建
   - 系统自动创建默认的收货、库存、发货、退货库位

2. **维护库位**
   - 库存 → 配置 → 库位 → 新建
   - 设置父库位、类型、条码、容量

3. **库位策略**
   - 配置上架规则 (Putaway Rules)：收货后自动建议存放位置
   - 配置拣货规则：按库位优先级排序拣货路径

---

### 2. 产品路线

#### 字段解释

| 字段 | 说明 |
|------|------|
| **路线 (Route)** | 定义产品从来源到目的地的移动路径和规则 |
| **规则 (Rule)** | 路线的具体步骤（如购买、制造、调拨） |
| **拉式规则** | 根据需求自动触发补货 |
| **推式规则** | 货物到达后自动推送到下一个库位 |
| **供应方法** | 从库存取货 / 触发另一规则 |

#### 标准路线

| 路线 | 说明 |
|------|------|
| **购买** | 缺货时自动生成采购订单 |
| **制造** | 缺货时自动生成生产订单 |
| **按订单购买 (MTO)** | 有销售订单才采购，不备库存 |
| **按库存生产 (MTS)** | 根据预测需求提前生产备货 |
| **内部调拨** | 仓库间补货 |
| **直运 (Dropship)** | 供应商直接发给客户，不经仓库 |

#### 配置步骤

1. **启用路线**
   - 库存 → 配置 → 设置：启用「多步路线」

2. **产品分配路线**
   - 产品 → 库存页签 → 路线：勾选适用的路线

3. **自定义路线**
   - 库存 → 配置 → 路线 → 新建
   - 添加规则，指定来源/目的库位、操作类型、供应方法

---

### 3. 操作类型

#### 字段解释

| 字段 | 说明 |
|------|------|
| **操作类型** | 库存移动的分类（收货、发货、内部调拨、制造领料等） |
| **序列号前缀** | 自动生成的单据编号前缀 |
| **源库位/目标库位** | 该操作的默认来源和去向 |
| **新建时显示操作** | 创建后是否直接进入操作界面 |

#### 标准操作类型

| 类型 | 说明 |
|------|------|
| **收货 (Receipts)** | 从供应商接收货物 |
| **内部调拨 (Internal)** | 仓库内部移动 |
| **发货 (Delivery Orders)** | 发给客户 |
| **退货 (Returns)** | 客户退货或退给供应商 |
| **废品 (Scrap)** | 报废处理 |
| **库存调整 (Inventory Adjustments)** | 盘点盈亏调整 |

---

### 4. 包装与计量单位

#### 计量单位

| 层级 | 说明 |
|------|------|
| **基本单位** | 产品管理的最小单位（如件、个、瓶） |
| **采购单位** | 采购时的计量单位（如箱=24瓶） |
| **销售单位** | 销售时的计量单位 |
| **换算率** | 不同单位间的换算关系 |

#### 包装

| 类型 | 说明 |
|------|------|
| **配送包装** | 运输用包装（如托盘、纸箱） |
| **销售包装** | 面向客户的包装规格 |
| **包含数量** | 每个包装包含的产品数量 |
| **重量/体积** | 用于运费计算 |

#### 配置步骤

- 产品 → 配置 → 计量单位：维护单位类别和换算率
- 产品 → 配置 → 包装：维护包装规格

## 二、库存作业

### 1. 收货 (Receipt)

#### 作业流程

```
采购订单确认 → 系统生成收货单 → 货物到达 → 扫描/核对 → 质检 → 上架入库
```

#### 字段解释

| 字段 | 说明 |
|------|------|
| **收货单号** | 系统自动生成 |
| **来源** | 关联的采购订单 |
| **收货行** | 产品、预期数量、已收数量 |
| **批次/序列号** | 收货时录入（如启用追溯） |
| **保质期** | 入库时录入到期日 |

#### 业务操作

- **查看待收货**：库存 → 作业 → 收货：查看所有待处理收货单
- **部分收货**：实际到货少于订单时，按实收数量确认
- **打印收货单**：仓库凭单核对
- **质检触发**：收货后自动生成质检单（如配置了质检点）

---

### 2. 上架 (Putaway)

#### 功能概览

将收到的货物按规则存放到合适的库位。

#### 上架策略

| 策略 | 说明 |
|------|------|
| **固定库位** | 每种产品有固定的存放位置 |
| **就近上架** | 放入距离最近的可用库位 |
| **按类别** | 同类产品放在同一区域 |
| **按属性** | 冷藏品放冷库，危险品放隔离区 |

#### 配置步骤

- 库存 → 配置 → 上架规则
- 按产品/产品类别/库位设置存放目标

#### 业务操作

- **系统建议**：收货确认后，系统根据上架规则建议目标库位
- **扫码上架**：扫描产品和库位条码，确认上架
- **批量上架**：同类商品批量分配到相邻库位

---

### 3. 内部调拨

#### 功能概览

在同一仓库或不同仓库间移动库存。

#### 业务场景

- 从收货区移动到存储区
- 从存储区移动到拣货区
- 从 A 仓库调拨到 B 仓库
- 从库位移到废品区报废

#### 业务操作

- **创建调拨**：库存 → 作业 → 内部调拨 → 新建
- **源库位/目标库位**：指定移动的起止位置
- **确认调拨**：移动完成后确认
- **跟踪**：查看产品的库位移动历史

---

### 4. 拣货 (Picking)

#### 功能概览

根据销售订单或生产订单，从库位拣选所需产品。

#### 拣货策略

| 策略 | 说明 |
|------|------|
| **单订单拣货** | 一次只拣一个订单 |
| **波次拣货** | 合并多个订单，按库位路径一次拣完 |
| **分区拣货** | 不同区域由不同人员同时拣货 |
| **先进先出 (FIFO)** | 优先拣最早入库的批次 |
| **后进先出 (LIFO)** | 优先拣最新入库的批次 |
| **先到期先出 (FEFO)** | 优先拣最早到期的批次 |

#### 配置步骤

- 仓库 → 配置 → 拣货策略
- 设置波次分组条件（如按承运商、按时间窗口）

#### 业务操作

- **查看拣货单**：库存 → 作业 → 拣货单
- **拣货路径**：系统按库位顺序排序，优化行走路线
- **扫码拣货**：扫描产品条码和库位条码确认
- **缺货处理**：库存不足时触发补货或拆单

---

### 5. 发货 (Delivery)

#### 作业流程

```
销售订单确认 → 系统生成拣货单 → 拣货完成 → 打包 → 生成发货单 → 发货 → 跟踪
```

#### 字段解释

| 字段 | 说明 |
|------|------|
| **发货单号** | 系统自动生成 |
| **承运商** | 快递公司/物流公司 |
| **跟踪号** | 物流跟踪单号 |
| **重量** | 包裹实际重量 |
| **包裹数** | 分拆成几个包裹 |
| **发货状态** | 草稿 → 等待可用 → 就绪 → 完成 |

#### 业务操作

- **自动发货**：订单确认后自动创建发货单
- **分批发货**：一个订单分多次发出
- **打印面单**：集成承运商 API 打印电子面单
- **发货通知**：发货后自动邮件通知客户
- **跟踪更新**：手动或自动更新物流跟踪状态

---

### 6. 退货处理

#### 客户退货

1. **创建退货单**
   - 在销售订单上点击「退货」
   - 选择退货产品和数量

2. **收货质检**
   - 仓库收到退货后检查状态
   - 合格品入库，不合格品报废或退回客户

3. **退款处理**
   - 财务根据退货单处理退款或换货

#### 退给供应商

- 在采购订单上创建「退给供应商」
- 生成反向收货单，库存减少

## 三、库存管理

### 1. 库存调整

#### 功能概览

当系统库存与实际库存不一致时，进行库存调整。

#### 调整原因

| 原因 | 说明 |
|------|------|
| **盘点盈亏** | 实际盘点与系统差异 |
| **损坏/丢失** | 产品损坏或丢失 |
| **入库差错** | 收货时录入错误 |
| **系统迁移** | 初始库存导入 |

#### 业务操作

- **创建调整**：库存 → 作业 → 库存调整 → 新建
- **调整行**：选择产品、库位、实际数量、调整原因
- **审批**：大额调整可能需要审批
- **过账**：确认后更新库存数量

---

### 2. 库存盘点

#### 盘点方式

| 方式 | 说明 |
|------|------|
| **年度大盘** | 年终对所有库存全面盘点 |
| **循环盘点** | 按 ABC 分类，高价值/高周转品频繁盘点 |
| **动碰盘点** | 每次出入库后随即盘点 |
| **抽样盘点** | 随机抽取部分库位盘点 |

#### 配置步骤

1. **创建盘点单**
   - 库存 → 作业 → 盘点 → 新建
   - 选择盘点范围（全部 / 按库位 / 按产品类别）

2. **设置盘点频率**
   - 产品 → 设置盘点频率（如 A 类每月，C 类每季）

3. **ABC 分类**
   - 按价值贡献分类：A（高价值，占 80% 金额）、B、C

#### 业务操作

- **打印盘点表**：导出盘点清单，仓库人员现场清点
- **扫码盘点**：使用条码终端扫描库位和产品，自动比对
- **差异处理**：盘点完成后查看差异报告，确认调整
- **锁库盘点**：盘点期间锁定相关库位禁止出入库

---

### 3. 库存计价

#### 计价方法

| 方法 | 说明 |
|------|------|
| **标准成本法** | 预设标准成本，差异单独分析 |
| **平均成本法 (AVCO)** | 每次入库后重新计算加权平均成本 |
| **先进先出 (FIFO)** | 按入库先后顺序出库 |
| **后进先出 (LIFO)** | 按入库相反顺序出库（部分国家禁用） |

#### 配置步骤

- 产品 → 产品类别 → 库存计价 → 选择成本方法
- 设置计价账户（存货科目、成本差异科目）

#### 业务操作

- **成本重估**：标准成本法下，定期调整标准成本
- **库存估值**：查看各产品的库存总价值
- **成本调整**：发现成本录入错误时进行调整

---

### 4. 批次与序列号

#### 批次 (Lot)

- 同一批生产/采购的产品共用同一个批次号
- 用于追溯、召回、保质期管理

#### 序列号 (Serial Number)

- 每个产品有唯一的序列号
- 用于高价值产品、保修管理、设备跟踪

#### 配置步骤

1. **启用追溯**
   - 产品 → 库存页签 → 追踪 → 选择「按批次」或「按序列号」

2. **录入追溯号**
   - 收货/生产入库时录入批次或序列号
   - 发货/领料时指定发出的批次/序列号

#### 业务操作

- **批次查询**：输入批次号查看所有相关移动记录
- **召回管理**：发现质量问题时，快速定位同批次所有产品
- **保质期预警**：按批次到期日预警即将过期产品

---

### 5. 保质期管理

#### 字段解释

| 字段 | 说明 |
|------|------|
| **到期日** | 产品过期日期 |
| **保质期天数** | 从生产到过期的天数 |
| **预警天数** | 到期前多少天开始预警 |
| **移除天数** | 到期前多少天禁止销售/使用 |

#### 配置步骤

- 产品 → 库存页签 → 启用保质期追踪
- 设置保质期和预警规则

#### 业务操作

- **FEFO 出库**：优先拣最早到期的批次
- **预警报表**：查看即将过期的批次清单
- **报废处理**：过期产品自动或手动移入废品区

## 四、条码作业

### 1. 条码配置

#### 功能概览

使用条码扫描终端（手机/平板/专业设备）进行无纸化仓库作业。

#### 配置步骤

1. **启用条码**
   - 库存 → 配置 → 设置：启用「条码扫描」

2. **产品条码**
   - 产品 → 基本信息 → 条码：录入 EAN/UPC/内部条码

3. **库位条码**
   - 库存 → 配置 → 库位 → 条码：为每个库位生成条码

4. **访问条码界面**
   - 在移动设备浏览器访问 Odoo 的条码应用
   - 或安装 Odoo Android/iOS APP

---

### 2. 收货条码作业

#### 操作流程

1. 扫描收货单条码（或手动选择）
2. 扫描产品条码
3. 输入/确认数量
4. 扫描目标库位条码
5. 确认上架

#### 业务操作

- **盲收模式**：不知道预期数量，按实际收货录入
- **按单收货**：按采购订单的预期明细逐项扫描确认
- **批次录入**：收货时扫描批次条码或手动输入批次号

---

### 3. 拣货条码作业

#### 操作流程

1. 扫描拣货单条码
2. 按系统提示走到指定库位
3. 扫描库位条码确认位置正确
4. 扫描产品条码确认品项
5. 输入实际拣货数量
6. 确认完成

#### 业务操作

- **语音拣货**：配合耳机和语音指令，解放双手
- **按波次拣货**：一次扫描处理多个订单的合并拣货单
- **差异处理**：实际库存不足时，系统提示缺货处理

---

### 4. 盘点条码作业

#### 操作流程

1. 选择盘点单
2. 扫描库位条码
3. 扫描该库位内所有产品条码并输入数量
4. 提交盘点结果
5. 系统自动比对生成差异报告

---

## 五、质量管理

### 1. 质检点配置

#### 字段解释

| 字段 | 说明 |
|------|------|
| **质检点** | 指定在何时/何地/对什么产品进行质检 |
| **类型** | 收货质检 / 出货质检 / 生产过程质检 / 在库质检 |
| **质检方式** | 全检 / 抽检（按百分比或固定数量） |
| **质检项目** | 外观 / 尺寸 / 功能 / 性能等检查项 |
| **判定标准** | 合格 / 让步接收 / 退货 / 报废 |

#### 配置步骤

1. **创建质检团队**
   - 质量 → 配置 → 团队

2. **设置质检点**
   - 质量 → 配置 → 质量控制点 → 新建
   - 指定产品/类别、操作类型、质检方式

3. **维护检查清单**
   - 为每个质检点设置详细的检查项和合格标准

---

### 2. 质检作业

#### 业务流程

```
收货/生产完成 → 自动触发质检单 → QC执行检查 → 判定结果 → 合格入库 / 不合格处理
```

#### 业务操作

- **查看待检**：质量 → 质检 → 待处理质检单
- **执行检查**：按检查清单逐项确认
- **拍照存档**：不合格品拍照记录
- **填写备注**：记录实测数据和观察

---

### 3. 不合格品处理

#### 处理方式

| 方式 | 说明 |
|------|------|
| **退货** | 退给供应商 |
| **报废** | 销毁处理 |
| **返工** | 退回生产返修 |
| **让步接收** | 降级使用，可能需客户同意 |

#### 业务操作

- **创建不合格报告 (NCR)**：记录不合格详情、原因分析
- ** corrective action**：发起纠正预防措施 (CAPA)
- **供应商索赔**：不合格品向供应商发起索赔

### 2. 运费计算

## 附录

### 库存 KPI 参考

| 指标 | 健康值 | 说明 |
|------|--------|------|
| 库存准确率 | > 98% | (1 - |差异|/总库存) |
| 拣货准确率 | > 99.5% | 正确订单/总订单 |
| 发货及时率 | > 95% | 按时发货订单占比 |
| 库存周转天数 | 行业相关 | 平均库存/(年销售成本/365) |
| 空间利用率 | > 80% | 已用库位/总库位 |

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 库存数量不对 | 未清采购/销售单据 | 检查草稿状态的出入库单 |
| 无法发货 | 库存不足或预留冲突 | 检查库存可用量、取消错误预留 |
| 成本不对 | 计价方法设置错误 | 检查产品类别的成本方法 |
| 条码扫不出 | 条码格式不匹配 | 确认条码位数和校验规则 |
