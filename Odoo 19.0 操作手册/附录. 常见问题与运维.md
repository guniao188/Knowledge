## 一、安装与部署

### 1. 系统要求

#### 最低配置

| 组件 | 要求 |
|------|------|
| **CPU** | 2 核 |
| **内存** | 4 GB RAM |
| **磁盘** | 20 GB SSD |
| **操作系统** | Ubuntu 22.04 LTS / Debian 11 / CentOS 8 |
| **Python** | 3.10+ |
| **PostgreSQL** | 14+ |
| **Node.js** | 18+ (前端构建) |

#### 推荐配置（生产环境）

| 规模 | CPU | 内存 | 磁盘 | 说明 |
|------|-----|------|------|------|
| 小型 < 50 人 | 4 核 | 8 GB | 100 GB SSD | 单服务器 |
| 中型 50-200 人 | 8 核 | 16 GB | 500 GB SSD | 分离 DB 和应用 |
| 大型 200+ 人 | 16 核+ | 32 GB+ | 1 TB+ SSD | 集群部署 |

---

### 2. 安装方式

| 方式 | 适用场景 | 优缺点 |
|------|---------|--------|
| **源码安装** | 开发/深度定制 | 灵活但复杂 |
| **Docker** | 快速部署/隔离环境 | 便捷但需了解容器 |
| **Odoo.sh** | 中小企业托管 | 官方托管，省心但费用 |
| **包管理器** | 测试/学习 | 简单但版本可能滞后 |

### 3. 数据库管理

#### 创建数据库

```bash
sudo -u postgres createdb -O odoo mydb
```

#### 删除数据库

```bash
sudo -u postgres dropdb mydb
```

#### 查看连接数

```sql
SELECT count(*) FROM pg_stat_activity;
```

## 二、备份与恢复

### 1. 数据库备份

#### 命令行备份

```bash
# 纯 SQL 备份
pg_dump -U odoo -h localhost mydb > backup.sql

# 自定义格式（推荐，支持并行恢复）
pg_dump -U odoo -h localhost -Fc mydb > backup.dump

# Odoo 内置备份
python odoo-bin dump mydb --format=zip
```

#### 自动备份

- 使用 cron 定时执行备份脚本
- 或使用 Odoo 自动备份模块

---

### 2. 文件存储备份

#### 文件存储位置

```
~/.local/share/Odoo/filestore/mydb/
```

#### 备份命令

```bash
rsync -avz ~/.local/share/Odoo/filestore/mydb/ /backup/filestore/
```

---

### 3. 灾难恢复

#### 恢复步骤

1. **停止 Odoo 服务**
2. **恢复数据库**
   ```bash
   pg_restore -U odoo -h localhost -d mydb backup.dump
   ```
3. **恢复文件存储**
   ```bash
   rsync -avz /backup/filestore/ ~/.local/share/Odoo/filestore/mydb/
   ```
4. **更新模块列表**
   ```bash
   python odoo-bin -u all -d mydb --stop-after-init
   ```
5. **启动服务**

## 三、性能优化

### 1. 数据库优化

#### PostgreSQL 配置

```ini
# /etc/postgresql/14/main/postgresql.conf
shared_buffers = 25% of RAM
effective_cache_size = 50% of RAM
work_mem = 256MB
maintenance_work_mem = 512MB
max_connections = 200
```

#### 数据库维护

```bash
# 分析表
sudo -u postgres psql -d mydb -c "ANALYZE;"

# 清理死元组
sudo -u postgres psql -d mydb -c "VACUUM ANALYZE;"

# 重建索引
sudo -u postgres psql -d mydb -c "REINDEX DATABASE mydb;"
```

### 2. 前端优化

#### 静态文件

- 启用 CDN 加速静态资源
- 压缩 CSS/JS 文件

### 3. 服务器配置

#### Nginx 反向代理

```nginx
upstream odoo {
    server 127.0.0.1:8069;
}

server {
    listen 80;
    server_name odoo.example.com;

    location / {
        proxy_pass http://odoo;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /longpolling {
        proxy_pass http://127.0.0.1:8072;
    }
}
```

## 五、故障排查

### 1. 服务无法启动

#### 排查步骤

1. 查看日志：`tail -f /var/log/odoo/odoo.log`
2. 检查端口占用：`netstat -tlnp | grep 8069`
3. 检查数据库连接：`psql -U odoo -h localhost -d mydb`
4. 检查配置文件语法

#### 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| Port already in use | 端口被占用 | 更换端口或杀死占用进程 |
| Connection refused | PostgreSQL 未启动 | 启动 PostgreSQL 服务 |
| Module not found | 模块依赖缺失 | 安装依赖模块 |

---

### 2. 页面加载慢

#### 排查步骤

1. 检查服务器资源（CPU、内存、磁盘 I/O）
2. 查看慢查询日志
3. 检查是否大量计算字段未存储
4. 检查网络延迟

#### 优化建议

- 启用多 worker 模式
- 增加 PostgreSQL 缓存
- 优化视图，减少字段数量
- 使用 CDN

---

### 3. 邮件发送失败

#### 排查步骤

1. 测试邮件服务器连接
   ```bash
   telnet smtp.gmail.com 587
   ```
2. 检查邮件服务器配置
3. 查看 Odoo 日志中的邮件错误
4. 检查是否被邮箱服务商拦截

#### 常见问题

| 问题 | 解决 |
|------|------|
| 认证失败 | 检查用户名密码，开启「允许不够安全的应用」 |
| 被标记为垃圾邮件 | 配置 SPF、DKIM、DMARC 记录 |
| 连接超时 | 检查防火墙和端口 |

---

### 4. 打印问题

#### 排查步骤

1. 检查 wkhtmltopdf 是否安装
   ```bash
   wkhtmltopdf --version
   ```
2. 检查报表模板是否有错误
3. 查看系统日志

#### 常见问题

| 问题 | 解决 |
|------|------|
| 中文乱码 | 安装中文字体包 |
| 样式丢失 | 检查 CSS 路径和权限 |
| 分页异常 | 调整 QWeb 模板的分页设置 |

---

### 5. 数据异常

#### 排查步骤

1. 检查是否有草稿状态的单据未处理
2. 查看审计日志追踪变更
3. 检查自动化规则是否误触发
4. 验证权限设置是否正确

#### 数据修复

- 使用 Odoo Shell 执行修复脚本
- 或联系 Odoo 官方支持

## 六、升级与迁移

### 1. 版本升级

#### 升级前准备

1. **完整备份**数据库和文件存储
2. **阅读升级说明**：了解破坏性变更
3. **测试环境验证**：先在测试环境升级

#### 升级步骤

1. 拉取新代码
2. 更新依赖：`pip install -r requirements.txt`
3. 更新数据库：`-u all`
4. 重启服务
5. 验证关键功能

---

### 2. 数据迁移

#### 场景

| 场景 | 方法 |
|------|------|
| 同版本迁移 | 备份恢复 |
| 跨版本迁移 | 使用 Odoo 升级服务或开源工具 |
| 其他 ERP 迁移 | 使用 ETL 工具或导入导出 |

## 七、常用命令

### 服务管理

```bash
# 启动 Odoo
python odoo-bin -c /etc/odoo/odoo.conf

# 更新模块
python odoo-bin -u all -d mydb --stop-after-init

# 安装模块
python odoo-bin -i sale -d mydb --stop-after-init

# 进入 Odoo Shell
python odoo-bin shell -d mydb

# 显示数据库列表
python odoo-bin --db-list
```

### 数据库

```bash
# 创建数据库
python odoo-bin -d mydb --init=base --stop-after-init

# 删除数据库
python odoo-bin --db-drop=mydb

# 备份数据库
python odoo-bin dump mydb --format=zip
```

### 日志查看

```bash
# 实时查看日志
tail -f /var/log/odoo/odoo.log

# 搜索错误
grep ERROR /var/log/odoo/odoo.log

# 查看最后 100 行
tail -n 100 /var/log/odoo/odoo.log
```
