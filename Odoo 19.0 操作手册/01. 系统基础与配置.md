## 一、系统初始化

### 1. 首次安装与数据库创建

#### 字段解释

| 字段 | 说明 |
|------|------|
| **数据库名称** | 唯一标识，建议使用英文小写 |
| **管理员邮箱** | 超级管理员账户，用于密码找回 |
| **管理员密码** | 建议 12 位以上，含大小写+数字+符号 |
| **语言** | 系统默认语言，影响界面和报表 |
| **国家** | 自动加载对应本地化模块 |
| **演示数据** | 勾选后填充示例数据，便于学习 |

#### 配置步骤

1. **访问安装页面**
   - 浏览器访问 `http://your-odoo-url/web/database/manager`
   - 或首次安装时自动跳转

2. **创建数据库**
   - 填写数据库名称
   - 选择是否加载演示数据（生产环境勿选）
   - 选择国家和语言
   - 设置管理员账户

3. **初始配置向导**
   - 登录后跟随「设置」向导完成基础配置
   - 公司信息、会计期间、币种、发票样式

> **警告**：生产环境务必关闭数据库管理界面，或在配置文件中设置 `list_db = False`。

### 2. 公司信息配置

#### 配置路径

设置 → 用户与公司 → 公司

#### 字段解释

| 字段 | 说明 |
|------|------|
| **公司名称** | 显示在单据抬头、报表、网站上 |
| **地址** | 发票、快递单上的公司地址 |
| **税号** | VAT/GST/税号，用于发票和报税 |
| **币种** | 公司本位币，影响所有金额显示 |
| **电话/邮箱/网站** | 联系信息，显示在对外单据上 |
| **Logo** | 显示在报表、网站、系统界面 |
| **银行账号** | 默认收款账户，显示在发票上 |
| **社会统计机构** | 社保、公积金等监管机构编号 |
| **会计科目表** | 加载的会计科目方案 |

#### 业务操作

- **多地址**：可维护总部、分公司、仓库等多个地址
- **公司印章**：上传电子签章图片用于签章模块
- **社交媒体**：配置后显示在网站页脚

### 3. 多公司管理

#### 功能概览

Odoo 支持在同一数据库中管理多个公司，数据可按公司隔离或共享。

#### 配置步骤

1. **启用多公司**
   - 设置 → 常规设置 → 多公司：勾选「多公司」

2. **创建子公司**
   - 设置 → 用户与公司 → 公司 → 新建
   - 指定父公司，形成集团架构

3. **公司间交易**
   - 安装 `account_inter_company_rules` 等模块（企业版）
   - 配置公司间的产品、客户映射

#### 业务操作

- **切换公司**：顶部导航栏点击公司名，可单选或多选可见公司
- **数据隔离**：各公司独立 seeing 自己的客户、订单、库存
- **合并报表**：在母公司视角查看合并的财务报表
- **共享数据**：产品、联系人可设置为「所有公司」可见

> **注意**：多公司配置应在上线初期确定，后期调整可能导致数据混乱。

### 4. 语言与区域设置

#### 配置步骤

1. **加载语言**
   - 设置 → 翻译 → 加载翻译：选择语言包
   - 勾选「覆盖现有术语」可更新已翻译内容

2. **用户个人语言**
   - 右上角头像 → 首选项 → 语言
   - 每位用户可独立设置界面语言

3. **文档翻译**
   - 表单视图右上角点击 🌐 图标
   - 逐字段填写多语言内容

#### 业务操作

- **翻译导出**：设置 → 翻译 → 导出翻译 → 选择语言和模块 → 下载 PO 文件
- **翻译导入**：翻译公司完成 PO 文件后导入
- **网站多语言**：网站 → 配置 → 设置 → 启用多语言 → 添加语言并翻译页面

### 5. 币种与汇率

#### 配置路径

会计 → 配置 → 币种 或 设置 → 币种

#### 字段解释

| 字段 | 说明 |
|------|------|
| **币种名称** | ISO 代码（CNY、USD、EUR 等） |
| **汇率** | 相对于本位币的兑换率 |
| **小数精度** | 金额显示的小数位 |
| **位置** | 货币符号在金额前/后 |
| **自动更新** | 通过欧洲央行自动获取汇率 |

#### 配置步骤

1. **激活币种**
   - 币种列表 → 勾选「活跃」启用需要的外币

2. **维护汇率**
   - 手动：在币种表单中直接输入汇率
   - 自动：会计 → 配置 → 自动汇率 → 设置更新频率和数据源

3. **汇率精度**
   - 设置 → 技术 → 小数精度：调整 `Account` 精度

> **待补充**：多币种交易的会计处理、未实现汇兑损益计算

## 二、用户与权限

### 1. 用户管理

#### 配置路径

设置 → 用户与公司 → 用户

#### 字段解释

| 字段 | 说明 |
|------|------|
| **名称** | 用户显示名称 |
| **登录邮箱** | 用户名，用于登录 |
| **访问权限** | 分配给用户的应用权限组 |
| **公司** | 用户所属公司 |
| **多公司** | 用户可访问的其他公司 |
| **通知方式** | 邮件 / Odoo 收件箱 / 从不 |
| **时区** | 影响日期时间显示 |
| **头像** | 显示在消息和讨论中 |
| **状态** | 新建 / 活跃 / 不活跃 |
| **API密钥** | 用于外部系统调用的访问令牌 |

#### 配置步骤

1. **创建用户**
   - 用户列表 → 新建 → 填写基本信息
   - 在「访问权限」页签勾选对应应用的角色
   - 发送邀请邮件（用户点击链接设置密码）

2. **批量邀请**
   - 设置 → 常规设置 → 用户 → 批量邀请
   - 粘贴邮箱列表，批量发送邀请

3. **停用用户**
   - 编辑用户 → 操作 → 归档
   - 归档用户不计入许可证数量

#### 业务操作

- **模拟用户**：管理员可「登录为」其他用户，排查权限问题
- **密码重置**：用户自助找回 或 管理员手动重置
- **权限预览**：在表单底部「已启用功能」区域查看当前用户的完整权限

---

### 2. 权限组 (Groups)

#### 功能概览

权限组是 Odoo 权限控制的核心，决定了用户能看到什么菜单、访问什么模型、执行什么操作。

#### 常见权限层级

| 角色 | 权限 | 适用人群 |
|------|------|---------|
| **仅自己文档** | 只能看到创建者是自己的记录 | 基层员工 |
| **自己文档 + 被指派** | 自己创建或被分配的记录 | 一般用户 |
| **全部文档** | 看到本公司的所有记录 | 部门主管 |
| **管理员** | 配置权限，看到所有数据 | 系统管理员 |

#### 配置步骤

1. **查看权限组**
   - 设置 → 用户与公司 → 组
   - 每个应用通常有：用户 / 管理员 / 高级用户 等组

2. **分配权限**
   - 用户表单 → 访问权限页签 → 勾选相应角色
   - 或进入权限组表单 → 添加用户

3. **继承权限**
   - 权限组可继承其他组（如「销售经理」自动包含「销售员」权限）
   - 技术设置中可查看继承链

> **待补充**：自定义权限组的创建方法、应用间权限交叉的影响

---

### 3. 访问控制列表 (ACL)

#### 功能概览

ACL 控制「哪个用户组」对「哪个模型」有什么权限（读/写/创建/删除）。

#### 配置路径（需开发者模式）

设置 → 技术 → 安全 → 访问控制列表

#### 字段解释

| 字段 | 说明 |
|------|------|
| **模型** | 受控的数据对象（如 res.partner、sale.order） |
| **组** | 适用的权限组 |
| **读取** | 查看记录 |
| **写入** | 修改记录 |
| **创建** | 新建记录 |
| **删除** | 删除记录 |

#### 业务操作

- **排查无权限**：用户报错「您没有访问此记录的权限」时，检查对应模型的 ACL
- **权限继承**：如果用户属于多个组，取各组权限的并集

---

### 4. 记录规则 (Record Rules)

#### 功能概览

记录规则在 ACL 基础上进一步过滤数据，实现「同一角色只能看到部分记录」的需求。

#### 配置路径（需开发者模式）

设置 → 技术 → 安全 → 记录规则

#### 常见规则

| 场景 | 规则表达式 |
|------|-----------|
| 仅看自己的客户 | `[('user_id', '=', user.id)]` |
| 仅看本公司数据 | `[('company_id', 'in', company_ids)]` |
| 仅看本团队订单 | `[('team_id.member_ids', 'in', [user.id])]` |

#### 业务操作

- **全局规则 vs 组规则**：全局规则对所有用户生效；组规则仅对该组成员生效
- **规则叠加**：多个规则同时生效时取交集

---

### 5. 安全与双因素认证

#### 配置步骤

1. **启用 2FA**
   - 设置 → 常规设置 → 登录 → 双因素认证
   - 用户个人设置中绑定 Authenticator APP

2. **密码策略**
   - 设置 → 常规设置 → 登录 → 最小密码长度、复杂度要求

3. **会话管理**
   - 设置 → 常规设置 → 会话：设置超时时间

4. **API 安全**
   - 用户 → 偏好 → 账户安全 → 生成 API 密钥
   - 定期轮换密钥，离职用户及时停用

## 三、邮件与消息

### 1. 邮件服务器配置

#### 配置路径

设置 → 技术 → 邮件 → 发件服务器 / 收件服务器

#### 发件服务器字段

| 字段 | 说明 |
|------|------|
| **描述** | 服务器标识 |
| **SMTP 服务器** | 如 smtp.gmail.com |
| **端口** | 25 / 465 (SSL) / 587 (TLS) |
| **连接安全** | 无 / SSL/TLS / STARTTLS |
| **用户名/密码** | 邮箱认证信息 |

#### 收件服务器 (IMAP)

| 字段 | 说明 |
|------|------|
| **服务器类型** | IMAP / POP |
| **服务器** | 如 imap.gmail.com |
| **端口** | 993 (IMAP SSL) |
| **动作** | 创建新记录 / 仅获取 |

#### 配置步骤

1. **配置发件服务器**
   - 填写 SMTP 信息 → 测试连接
   - 可配置多个服务器，按优先级轮询

2. **配置收件服务器**
   - 绑定到特定模型（如 CRM 线索、服务台工单）
   - 设置抓取间隔

3. **系统邮件地址**
   - 设置 → 常规设置 → 别名域：配置默认发件域名

---

### 2. 别名系统 (Aliases)

#### 功能概览

通过特定邮箱地址自动创建业务记录，如 `sales@company.com` 自动创建商机。

#### 配置路径

设置 → 技术 → 邮件 → 别名

#### 常见别名

| 别名 | 功能 |
|------|------|
| `sales@` | 创建 CRM 商机 |
| `support@` | 创建服务台工单 |
| `project+xxx@` | 在指定项目中创建任务 |
| `catchall@` | 退回邮件的接收地址 |
| `bounce@` | 邮件退信处理 |

#### 配置步骤

1. **设置别名域**
   - 设置 → 常规设置 → 讨论 → 别名域
   - 如 `company.com`

2. **配置具体别名**
   - 进入目标应用（如 CRM 销售团队）
   - 在团队表单中设置「邮箱别名」

3. **MX 记录**
   - 在 DNS 中将别名域指向 Odoo 服务器

---

### 3. 邮件模板

#### 配置路径

设置 → 技术 → 邮件 → 模板

#### 字段解释

| 字段 | 说明 |
|------|------|
| **名称** | 模板标识 |
| **模型** | 适用的业务对象 |
| **主题** | 邮件标题，支持动态变量 |
| **收件人** | 自动填充的邮箱字段 |
| **正文** | 支持 HTML 和 Jinja2 变量 |
| **报告** | 附件的报表模板 |

#### 业务操作

- **动态变量**：使用 `${object.name}` 引用记录字段
- **预览**：选择具体记录预览邮件渲染效果
- **翻译**：为不同语言用户发送对应语言版本
- **条件显示**：使用 Jinja2 的 `{% if %}` 控制内容显示

---

### 4. 消息通知策略

#### 配置路径

用户个人首选项 → 通知

#### 选项

| 方式 | 说明 |
|------|------|
| **邮件** | 每条消息都发邮件 |
| **Odoo 收件箱** | 仅在系统内通知 |
| **从不** | 不接收任何通知 |
| **每日摘要** | 汇总一天的消息 |
| **每周摘要** | 汇总一周的消息 |

#### 业务操作

- **关注/取消关注**：在业务记录上点击「关注」接收动态通知
- **@提及**：消息中 `@用户名` 强制通知该用户
- **频道通知**：Discuss 频道可设为静音或提醒所有消息

## 四、通用界面操作

### 1. 视图类型

| 视图 | 用途 | 操作 |

#### 视图切换

- 列表/看板/日历/数据透视等视图通过右上角图标切换
- 部分视图需特定模块支持

---

### 2. 搜索与筛选

#### 搜索功能

- **快速搜索**：顶部搜索框输入关键词
- **筛选器**：点击漏斗图标选择预设条件
- **分组依据**：按字段分组显示
- **收藏**：将常用筛选条件保存为个人或公共收藏

#### 高级搜索

- **添加自定义筛选**：指定字段、操作符、值
- **组合条件**：AND / OR 逻辑组合
- **日期范围**：今天、本周、本月、自定义区间

---

### 3. 收藏与仪表盘

#### 收藏功能

- **保存当前搜索**：筛选 + 分组 → 收藏 → 保存
- **设为默认**：将收藏设为进入该视图时的默认显示
- **共享**：设为公共收藏供团队使用

#### 仪表盘

- **添加报表到仪表盘**：在报表视图 → 收藏 → 添加到仪表盘
- **仪表盘编辑**：拖拽调整位置、选择图表类型
- **定时发送**：设置自动邮件推送仪表盘

## 五、数据管理

### 1. 导入数据

#### 字段解释

| 字段 | 说明 |
|------|------|
| **外部 ID** | 数据唯一标识，用于更新匹配 |
| **列映射** | Excel 列与 Odoo 字段对应 |
| **导入模式** | 仅新建 / 仅更新 / 新建并更新 |
| **预览** | 导入前的校验结果 |

#### 配置步骤

1. **准备模板**
   - 目标列表 → 导出 → 勾选「我要更新数据」→ 导出外部 ID
   - 或下载系统提供的导入模板

2. **执行导入**
   - 收藏 → 导入记录 → 上传文件
   - 系统识别列头 → 调整映射 → 预览校验
   - 处理错误 → 确认导入

#### 导入技巧

- **多对一**：填写关联记录名称或外部 ID
- **多对多**：逗号分隔多个值
- **日期**：使用 `YYYY-MM-DD` 格式
- **布尔值**：`1`/`0` 或 `True`/`False`
- **父子关系**：使用「父级外部 ID」字段

> **注意**：首次导入建议先在测试环境验证，生产环境导入前备份数据库。

---

### 2. 导出数据

#### 配置步骤

1. **选择字段**
   - 列表视图 → 收藏 → 导出
   - 勾选需要导出的字段
   - 可导出关联字段（如订单的客户名称）

2. **格式选择**
   - Excel (.xlsx)
   - CSV

3. **导出外部 ID**
   - 勾选「我要更新数据」以导出外部 ID，便于后续修改后重新导入

---

### 3. 数据清洗

#### 配置路径

设置 → 数据清洗 或 数据清洗应用

#### 功能

- **重复检测**：按配置规则扫描疑似重复记录
- **相似度排序**：按算法匹配结果排序
- **智能合并**：选择主记录，合并字段冲突
- **批量归档**：批量废弃不需要的记录

#### 配置步骤

1. **启用检测**
   - 数据清洗 → 配置 → 模型
   - 选择要监控的模型和匹配字段

2. **设置规则**
   - 配置匹配算法（如联系人按名称+邮箱相似度匹配）
   - 设置自动执行频率

- 支持图片预览、PDF 在线查看

参见 [[09. 生产力工具]] → 文档管理

## 六、系统维护

### 1. 模块安装与卸载

#### 安装模块

1. **应用菜单**
   - 应用 → 搜索模块名称
   - 点击「激活」安装

2. **依赖自动处理**
   - 系统会自动安装依赖模块
   - 安装后可能需要刷新页面

3. **配置向导**
   - 部分模块安装后会弹出配置向导

#### 卸载模块

- 应用 → 找到模块 → 表单右上角 ⋮ → 卸载
- > **警告**：卸载模块会删除该模块创建的数据，请谨慎操作！

---

### 2. 系统更新

#### 应用更新

- 应用 → 点击更新列表 → 可更新的模块会显示更新按钮
- 或直接命令行执行 `-u all`

#### 代码更新

1. 拉取最新代码
2. 更新数据库：`-u all` 或指定模块
3. 重启 Odoo 服务

---

### 3. 日志与调试

#### 开发者模式

- 设置 → 常规设置 → 最下方「激活开发者模式」
- 或 URL 后加 `?debug=1`

#### 开发者工具

| 功能 | 说明 |
|------|------|
| **视图元信息** | 查看当前视图的 XML 架构 |
| **管理过滤器** | 查看/编辑保存的搜索条件 |
| **翻译** | 直接编辑界面术语翻译 |
| **审计日志** | 查看字段的历史变更记录 |
| **技术菜单** | 访问模型、视图、规则、计划任务等 |

#### 日志级别

- `DEBUG`：详细调试信息
- `INFO`：一般信息
- `WARNING`：警告
- `ERROR`：错误
