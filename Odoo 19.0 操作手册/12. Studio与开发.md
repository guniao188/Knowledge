## 一、Studio 入门

### 1. 启动 Studio

#### 方式

- 任何界面点击右上角「🛠️ Studio」图标
- 应用菜单 → Studio → 新建应用

> **要求**：需要用户具有「Studio」权限组。

---

### 2. 创建新应用

#### 步骤

1. Studio → 新建应用
2. 输入应用名称（如「设备报修」）
3. 选择基础模板：
   - 空白：从零开始
   - CRM 风格：类似 CRM 的漏斗流程
   - 项目风格：类似项目的任务管理
4. 系统自动创建：菜单、模型、默认视图、权限组

---

### 3. Studio 界面介绍

| 区域 | 功能 |
|------|------|
| **顶部工具栏** | 保存、撤销、预览、导出 |
| **左侧面板** | 字段库、视图选择、组件 |
| **中间画布** | 可视化编辑区域 |
| **右侧面板** | 选中元素的属性配置 |
| **底部状态栏** | 当前模型和技术信息 |

## 二、模型与字段

### 1. 创建模型

#### 字段解释

| 字段 | 说明 |
|------|------|
| **模型名称** | 系统标识（如 x_equipment_request） |
| **显示名称** | 用户看到的名称 |
| **模型类型** | 普通模型 / 邮件继承 / 活动继承 |

---

### 2. 字段类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **文本** | 单行文本 | 名称、标题 |
| **多行文本** | 多行文本框 | 描述、备注 |
| **整数** | 整数值 | 数量 |
| **小数** | 浮点数 | 金额、重量 |
| **日期** | 日期选择器 | 日期 |
| **日期时间** | 含时间的日期 | 创建时间 |
| **布尔** | 勾选框 | 是否 |
| **选择** | 下拉单选 | 状态、优先级 |
| **多选** | 下拉多选 | 标签 |
| **文件** | 附件上传 | 文档 |
| **图片** | 图片上传 | 头像、照片 |
| **HTML** | 富文本编辑 | 内容 |

---

### 3. 字段属性

| 属性 | 说明 |
|------|------|
| **必填** | 保存时必须填写 |
| **只读** | 不可编辑 |
| **唯一** | 值不能重复 |
| **索引** | 加快搜索速度 |
| **默认值** | 新建时的默认值 |
| **帮助提示** | 字段旁的说明文字 |
| **可见性条件** | 满足条件时才显示 |
| **只读条件** | 满足条件时变为只读 |

---

### 4. 关系字段

| 类型 | 说明 |
|------|------|
| **多对一 (Many2one)** | 关联到另一个模型的单条记录，如「客户」 |
| **一对多 (One2many)** | 关联模型的多条记录，如「订单行」 |
| **多对多 (Many2many)** | 双向多选关系，如「标签」 |
| **属性字段 (Properties)** | 动态的、按记录存储的扩展字段 |

## 三、视图定制

### 1. 表单视图

#### 业务操作

- **拖拽字段**：从左侧面板拖拽字段到画布
- **调整布局**：拖拽调整字段位置
- **添加组件**：
  - Notebook（标签页）
  - Group（分组框）
  - HTML 字段（富文本）
  - 智能按钮（统计关联记录数量）
- **设置属性**：选中元素后在右侧面板配置

---

### 2. 列表视图

#### 业务操作

- **选择显示列**：勾选要在列表中显示的字段
- **设置排序**：默认按哪个字段排序
- **可编辑列表**：直接在列表中编辑单元格
- **冻结列**：左侧列固定不动

---

### 3. 看板视图

#### 业务操作

- **设置分组**：按哪个字段分组（通常是状态/阶段）
- **设计卡片**：拖拽字段到卡片模板
- **设置颜色**：按字段值显示不同颜色
- **进度条**：显示完成进度

---

### 4. 搜索视图

#### 业务操作

- **添加筛选器**：预设常用筛选条件
- **添加分组依据**：快速分组选项
- **自动完成**：设置搜索建议字段

---

### 5. 其他视图

| 视图 | 用途 |
|------|------|
| **日历** | 按日期展示记录 |
| **甘特** | 时间线排期 |
| **数据透视** | 多维分析 |
| **图表** | 可视化统计 |
| **地图** | 地理分布 |
| **活动** | 待办跟踪 |

## 四、自动化与流程

### 1. 自动化规则

#### 字段解释

| 字段 | 说明 |
|------|------|
| **触发条件** | 创建 / 更新 / 删除 / 定时触发 |
| **适用模型** | 作用于哪个数据对象 |
| **筛选条件** | 仅在满足条件时触发 |
| **执行动作** | 更新字段 / 创建记录 / 发送邮件 / 添加关注者 |

#### 配置步骤

- Studio → 自动化 → 自动化规则 → 新建
- 设置触发条件和筛选器
- 添加执行动作

#### 业务场景

- **自动分配**：新线索自动分配给销售
- **状态联动**：订单确认后自动更新客户状态
- **逾期提醒**：发票逾期 7 天自动发送催款邮件

---

### 2. 邮件模板自动化

#### 配置步骤

- Studio → 自动化 → 邮件模板
- 设置触发条件和收件人
- 设计邮件内容（支持 HTML 和动态变量）

---

### 3. 计划动作

#### 功能概览

定时执行的后台任务。

#### 配置步骤

- 设置 → 技术 → 自动化 → 计划动作
- 设置执行频率（每小时/每天/每周）
- 指定要执行的 Python 代码或方法

---

### 4. 审批流程

#### 配置步骤

- Studio → 自动化 → 审批流程
- 设置审批层级和条件
- 指定审批人和超时规则

#### 审批配置

| 设置 | 说明 |
|------|------|
| **审批人** | 指定用户 / 上级经理 / 角色 |
| **条件** | 金额 > 10000 才需要该级审批 |
| **超时** | 24 小时未审批自动升级 |
| **委托** | 审批人请假时委托他人 |

## 五、报表设计

### 1. QWeb 报表

#### 技术说明

QWeb 是 Odoo 的模板引擎，基于 XML + HTML 语法。

#### 基本结构

```xml
<t t-name="report_custom">
  <div class="page">
    <h1>Report Title</h1>
    <t t-foreach="docs" t-as="doc">
      <p><span t-field="doc.name"/></p>
    </t>
  </div>
</t>
```

#### 常用指令

| 指令 | 说明 |
|------|------|
| `t-foreach` | 循环 |
| `t-if` | 条件判断 |
| `t-field` | 显示字段值 |
| `t-esc` | 转义输出 |
| `t-raw` | 原样输出 HTML |
| `t-call` | 调用子模板 |

---

### 2. 打印模板

- Studio → 报表 → 新建

### 2. API 接口

#### XML-RPC

```python
import xmlrpc.client

url = 'http://localhost:8069'
db = 'mydb'
username = 'admin'
password = 'admin'

common = xmlrpc.client.ServerProxy(f'{url}/xmlrpc/2/common')
uid = common.authenticate(db, username, password, {})

models = xmlrpc.client.ServerProxy(f'{url}/xmlrpc/2/object')
models.execute_kw(db, uid, password, 'res.partner', 'search_read', [[['is_company', '=', True]]], {'fields': ['name'], 'limit': 5})
```

#### JSON-RPC

```bash
curl -X POST http://localhost:8069/jsonrpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "method": "call", "params": {"service": "object", "method": "execute", "args": ["mydb", 1, "admin", "res.partner", "search_read", [[["is_company", "=", true]]]]}, "id": 1}'
```

#### REST API (第三方模块)

部分社区模块提供 RESTful API 封装。

## 七、Studio 进阶

### 1. 导出模块

#### 业务操作

- Studio → 导出 → 选择要导出的组件
- 生成 ZIP 文件，包含所有自定义的模型、视图、数据
- 可在其他 Odoo 实例中导入

---

### 2. 与代码开发协作

#### 协作边界

| Studio 适合 | 代码开发适合 |
|------------|------------|
| 添加字段和视图 | 复杂业务逻辑 |
| 简单自动化 | 自定义工作流引擎 |
| 报表模板调整 | 深度系统集成 |
| 审批流程配置 | 大规模数据迁移 |

---

### 3. 性能注意事项

- **字段数量**：单个模型字段过多会影响加载速度
- **计算字段**：避免复杂计算字段，或设置存储
- **视图嵌套**：避免过深的视图继承链
- **自动化规则**：避免循环触发
