## 一、供应商管理

### 1. 供应商档案

#### 功能概览

供应商在 Odoo 中作为「联系人 (res.partner)」的一种类型存在，标记「供应商」复选框即可启用采购相关功能。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **名称** | 公司全称 |
| **供应商等级** | 用于筛选和优先级排序 |
| **地址** | 主要办公地址和账单地址 |
| **联系人** | 采购对接人 |
| **税号** | 增值税号或统一社会信用代码 |
| **银行账号** | 付款账户 |
| **付款条款** | 默认的付款账期 |
| **交货条款** | Incoterm（如 FOB、CIF） |
| **币种** | 交易默认币种 |
| **供应商标签** | 分类标签（如原材料、办公用品、服务） |

#### 配置步骤

1. **创建供应商**
   - 采购 → 订单 → 供应商 → 新建
   - 填写公司信息，勾选「供应商」类型

2. **设置默认值**
   - 在供应商卡片上设置默认的付款条款、交货条款、币种
   - 这些默认值在创建采购订单时自动带入

3. **多地址管理**
   - 可维护多个收货地址（如不同工厂）
   - 多个账单地址（如不同法人实体）

#### 业务操作

- **快速建档**：在采购订单的供应商字段直接输入名称，回车快速创建
- **供应商合并**：发现重复供应商时使用数据清洗工具合并
- **禁用供应商**：取消勾选「供应商」或归档，禁止新的采购订单

---

### 2. 供应商评估

#### 功能概览

通过统计采购数据和手动评分，综合评估供应商的交付能力、质量、价格竞争力。

#### 评估维度

| 维度 | 指标 |
|------|------|
| **交付准时率** | 按时交货订单占比 |
| **质量合格率** | 免检/一次合格占比 |
| **价格竞争力** | 与市场均价对比 |
| **响应速度** | 报价响应时间、问题处理时效 |
| **配合度** | 售后支持、紧急订单配合 |

#### 配置步骤

1. **启用供应商评价**
   - 采购 → 配置 → 设置：启用「供应商评价」

2. **设置评分标准**
   - 维护评分维度和权重

#### 业务操作

- **自动评分**：系统根据交货准时率、退货率自动生成部分评分
- **手动评分**：采购员按周期对供应商各维度打分
- **供应商分级**：A/B/C/D 级，A 级优先下单，D 级淘汰或整改

---

### 3. 供应商价格表

#### 功能概览

维护每个供应商针对每个产品的采购价格，支持多币种、批量折扣、有效期管理。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **供应商** | 价格提供方 |
| **产品** | 对应物料 |
| **产品变体** | 如适用特定变体 |
| **数量** | 达到该数量适用的价格 |
| **单价** | 采购单价 |
| **币种** | 报价币种 |
| **有效期** | 价格生效起止日期 |
| **交货提前期** | 下单到交货的工作日数 |

#### 配置步骤

1. **录入供应商价格**
   - 产品 → 产品 → 采购页签 → 添加供应商
   - 或在采购 → 配置 → 供应商价格表中批量维护

2. **导入批量价格**
   - 使用导入模板批量更新价格
   - 建议包含有效期以便价格到期自动失效

#### 业务操作

- **价格比较**：在采购申请时查看多个供应商的报价对比
- **历史价格**：查看同一产品历史采购价格趋势
- **价格预警**：价格波动超过阈值时通知采购员

## 二、采购订单

### 1. 采购申请 (PR)

#### 功能概览

采购申请是内部部门向采购部门提出的采购需求，需经审批后转为正式询价。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **申请部门** | 提出需求的部门 |
| **申请人** | 需求提出人 |
| **需求日期** | 物料需要的日期 |
| **产品/描述** | 需要采购的物品 |
| **数量** | 需求数量 |
| **预估单价** | 申请人预估的价格 |
| **用途说明** | 采购理由和用途 |
| **审批状态** | 草稿 → 待审批 → 已批准 → 已拒绝 |

#### 配置步骤

1. **启用采购申请**
   - 采购 → 配置 → 设置：启用「采购申请」

2. **设置审批流**
   - 按金额设置审批层级（如 <5000 部门经理批，>5000 总监批）

#### 业务操作

- **提交申请**：需求部门填写采购申请 → 提交审批
- **审批处理**：审批人在待办中查看并批准/拒绝
- **转为询价**：采购员将批准的申请合并为询价单
- **紧急采购**：紧急需求可标记加急，缩短审批时限

---

### 2. 询价单 (RFQ)

#### 功能概览

询价单是向供应商发出的价格询问，可同一需求向多家供应商询价，比价后选择最优方案转为采购订单。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **供应商** | 询价对象 |
| **参考号** | 供应商的报价参考号 |
| **订单行** | 产品、数量、供应商报价 |
| **预计交货** | 供应商承诺的交货日期 |
| **条款** | 付款条款、交货条款 |
| **状态** | 草稿 → 已发送 → 已确认 → 已取消 |

#### 业务操作

- **创建询价**：从采购申请转化 / 手动新建
- **发送询价**：点击「发送邮件」向供应商发送 PDF 询价单
- **录入报价**：收到供应商回复后在订单行录入供应商报价
- **比价**：在列表视图同时查看多个供应商的报价
- **确认订单**：选择最优报价，点击「确认订单」转为 PO
- **拒绝其他**：向未中标的供应商发送感谢信

### 3. 采购订单 (PO)

#### 订单生命周期

```
采购申请 → 询价单 → 采购订单 → 收货 → 质检 → 入库 → 收票 → 付款
```

#### 字段解释

| 字段 | 说明 |
|------|------|
| **订单号** | 系统自动生成的唯一编号 |
| **供应商** | 供货方 |
| **订单日期** | PO 创建日期 |
| **预期到货** | 计划收货日期 |
| **目的地** | 收货仓库地址 |
| **订单行** | 产品、数量、单价、小计 |
| **小计/税额/总计** | 自动计算 |

#### 业务操作

- **确认订单**：采购员确认后发送给供应商
- **打印 PO**：生成 PDF 格式的正式采购订单
- **跟踪到货**：在订单列表查看「已收/已开票」状态
- **分批收货**：一个 PO 可分多次收货
- **取消订单**：未收货前可取消，已部分收货的需处理后续

---

### 4. 收货与入库

#### 功能概览

采购订单确认后，仓库按到货通知执行收货、质检、入库操作。

参见 [[04. 库存与物流]] → 收货作业，获取详细的入库操作流程。

#### 采购关联字段

| 字段 | 说明 |
|------|------|
| **采购订单关联** | 收货单自动关联对应的 PO |
| **已收数量** | PO 上显示累计收货数量 |
| **待收数量** | 订单数量 - 已收数量 |
| **超收处理** | 收货超过订单数量时的策略 |

---

### 5. 供应商账单

#### 功能概览

收到供应商发票后，在系统中登记账单，与采购订单和收货单进行匹配。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **供应商** | 开票方 |
| **账单日期** | 发票日期 |
| **到期日** | 付款截止日期 |
| **参考号** | 供应商发票号码 |
| **采购订单** | 关联的 PO |
| **订单行** | 自动带出 PO 明细，可修改 |
| **金额匹配** | 账单金额 vs 订单金额 vs 收货金额 |

#### 业务操作

- **创建账单**：从 PO 点击「创建账单」自动带出明细
- **三方匹配**：系统比对 PO 金额、收货数量、发票金额是否一致
- **差异处理**：价格/数量差异时，按预设规则处理（拒绝/警告/通过）
- **审批与过账**：账单确认后生成会计分录
- **付款申请**：过账后进入付款流程

## 三、采购协议与招标

### 1. 采购协议（框架协议）

#### 功能概览

与核心供应商签订长期框架协议，约定价格、数量、交货条款，后续按需下达具体订单。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **协议类型** | 一揽子订单 / 招标 / 独家采购 |
| **供应商** | 协议签约方 |
| **有效期** | 协议生效起止日期 |
| **产品/类别** | 协议覆盖的产品范围 |
| **协议价格** | 锁定的采购单价 |
| **最小采购量** | 协议承诺的采购数量 |
| **已执行量** | 累计已下单数量 |
| **状态** | 草稿 → 确认 → 进行中 → 完成 → 取消 |

#### 配置步骤

1. **创建协议**
   - 采购 → 协议 → 新建
   - 选择协议类型和供应商
   - 添加协议产品行和价格条款

2. **设置提醒**
   - 协议到期前自动提醒续签
   - 执行量接近上限时预警

#### 业务操作

- **下达订单**：在协议上点击「新建报价/订单」自动带入协议价格
- **执行跟踪**：实时查看协议执行进度
- **续签管理**：到期前启动续签谈判流程

---

### 2. 招标管理

#### 功能概览

针对大额或重要采购，通过公开招标邀请多家供应商竞价，确保采购透明和成本最优。

#### 字段解释

| 字段 | 说明 |
|------|------|
| **招标编号** | 唯一标识 |
| **招标名称** | 项目描述 |
| **招标类型** | 公开 / 邀请 |
| **截标日期** | 投标截止时间 |
| **开标日期** | 评标时间 |
| **投标商** | 参与竞标的供应商 |
| **评标标准** | 价格权重、质量权重、交付权重 |
| **中标商** | 最终选定的供应商 |

#### 配置步骤

1. **创建招标**
   - 采购 → 招标 → 新建
   - 填写招标需求、技术规格、商务条款

2. **邀请投标**
   - 向合格供应商库发送投标邀请
   - 供应商通过门户提交投标文件

3. **评标与定标**
   - 评标委员会打分
   - 系统按预设权重自动计算综合得分
   - 确定中标商，生成中标通知

#### 业务操作

- **投标保密**：各供应商只能看到自己的报价
- **多轮谈判**：支持多轮报价和谈判记录
- **审计追踪**：完整记录招标过程和决策依据

## 四、采购分析

### 1. 采购报表

#### 报表类型

| 报表 | 说明 |
|------|------|
| **采购金额分析** | 按供应商/产品/时间段汇总采购金额 |
| **价格趋势** | 同一产品历史采购价格走势 |
| **交货准时率** | 供应商按时交货统计 |
| **采购周期** | 从申请到收货的平均周期 |
| **库存周转** | 采购量 vs 销售量 vs 库存量 |

#### 业务操作

- **数据透视**：多维度分析采购结构
- **图表展示**：柱状图对比各供应商采购额
- **导出**：Excel / PDF 导出用于汇报

---

### 2. 供应商绩效

#### KPI 指标

| 指标 | 计算方式 |
|------|---------|
| **准时交货率** | 准时交货订单数 / 总订单数 |
| **质量合格率** | 合格收货批次 / 总收货批次 |
| **价格竞争力** | 本供应商价 / 市场均价 |
| **响应速度** | 平均报价响应小时数 |
| **配合度评分** | 采购员主观评分 |

#### 业务操作

- **自动计算**：系统根据订单和收货数据自动计算客观指标
- **定期评审**：每季度/年度生成供应商绩效报告
- **分级调整**：根据绩效结果调整供应商等级和采购份额

## 五、高级功能

### 1. 一揽子订单

#### 功能概览

Blanket Order，与供应商约定一段时期内的总采购量和价格，具体交货日期分批确认。

#### 业务操作

- **创建一揽子订单**：约定总量、价格、有效期
- **分批释放**：根据生产/销售计划，定期释放具体交货指令
- **执行跟踪**：累计释放量 vs 总量上限

---

### 2. 采购审批流

#### 配置步骤

1. **设置审批规则**
   - 采购 → 配置 → 审批规则
   - 按金额、产品类别、供应商等级设置审批层级

2. **审批人配置**
   - 指定各层级的审批人或审批组
   - 支持代理审批（审批人请假时）

#### 审批层级示例

| 金额区间 | 审批人 |
|---------|--------|
| < 5,000 | 部门经理 |
| 5,000 - 50,000 | 采购总监 |
| > 50,000 | CFO + CEO 联签 |

---

### 3. 三方匹配

#### 功能概览

将采购订单 (PO)、收货单 (Receipt)、供应商账单 (Vendor Bill) 三方数据进行自动比对，确保一致后才允许付款。

#### 匹配规则

| 匹配项 | 说明 |
|--------|------|
| **数量匹配** | 账单数量 = 收货数量（允许容差） |
| **价格匹配** | 账单单价 = PO 单价（允许容差） |
| **总额匹配** | 账单总额 = PO 总额（允许容差） |

#### 配置步骤

- 采购 → 配置 → 设置 → 三方匹配
- 设置容差范围（如 ±1% 或 ±100 元）
- 不匹配时的处理策略：阻止付款 / 仅警告

## 附录

### 采购流程检查清单

#### 采购前

- [ ] 需求已确认，规格清晰
- [ ] 预算已获批
- [ ] 已进行供应商比价（大额采购）
- [ ] 框架协议已执行（如适用）

#### 采购中

- [ ] 采购订单已发送给供应商并确认收到
- [ ] 交货日期已确认并在系统中更新
- [ ] 紧急订单已标记并通知相关方

#### 采购后

- [ ] 收货数量与 PO 一致（或在容差内）
- [ ] 质检合格，已入库
- [ ] 供应商账单已核对并过账
- [ ] 付款已排期或已执行
- [ ] 供应商绩效已记录

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 无法选择供应商 | 联系人未勾选「供应商」类型 | 编辑联系人，勾选供应商 |
| 采购订单无法确认 | 产品未设置采购路线 | 产品 → 库存 → 路线，启用「购买」 |
| 账单金额与 PO 不符 | 价格变动或数量差异 | 检查三方匹配容差设置 |
| 找不到采购申请菜单 | 未启用采购申请功能 | 采购 → 配置 → 设置 → 启用采购申请 |
