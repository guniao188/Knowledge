## 一、项目管理

### 1. 项目创建

#### 字段解释

| 字段 | 说明 |
|------|------|
| **项目名称** | 项目标识 |
| **客户** | 关联的客户（可选） |
| **项目经理** | 项目负责人 |
| **标签** | 用于分类筛选 |
| **可见性** | 私密 / 内部 / 门户 |
| **账单类型** | 不可计费 / 按任务计费 / 按工时计费 / 按里程碑计费 |
| **预计收入** | 项目预算收入 |
| **预计成本** | 项目预算成本 |

#### 配置步骤

- 项目 → 新建 → 填写项目信息
- 设置项目团队（成员、关注者）
- 配置账单方式和价格表（如按人天计费）

---

### 2. 任务管理

#### 字段解释

| 字段 | 说明 |
|------|------|
| **任务名称** | 任务描述 |
| **分配对象** | 负责人 |
| **截止日期** | 计划完成日期 |
| **优先级** | 低 / 中 / 高 / 紧急 |
| **标签** | 分类标签 |
| **阶段** | 任务状态 |
| **预估工时** | 计划投入时间 |
| **实际工时** | 已记录工时 |
| **父任务** | 上级任务（子任务关系） |

#### 业务操作

- **创建任务**：直接新建 / 从邮件创建 / 从销售订单创建
- **看板协作**：拖拽任务卡片变更阶段
- **子任务**：将大任务拆分为多个子任务
- **任务依赖**：设置前置任务，前置未完成时提醒
- **邮件集成**：发送邮件到项目别名自动创建任务
- **文档关联**：在任务中上传交付物、关联知识库

---

### 3. 项目阶段

#### 配置步骤

- 项目 → 配置 → 阶段 → 新建
- 设置阶段名称、适用项目、折叠状态
- 设置「邮件模板」（进入该阶段时自动发送通知）

---

### 4. 时间管理

#### 功能

- **甘特图**：可视化任务排期，拖拽调整时间
- **日历视图**：按日历查看任务和里程碑
- **里程碑**：标记关键节点日期

---

### 5. 项目成本与收入

#### 收入确认

| 方式 | 说明 |
|------|------|
| **按工时计费** | 记录工时后按费率生成发票 |
| **按任务计费** | 任务完成后按固定价格开票 |
| **按里程碑** | 达成里程碑后开票 |
| **固定价格** | 项目总价固定，按约定节点开票 |

#### 成本跟踪

- 人工成本：工时 × 人工费率
- 采购成本：项目相关的采购费用
- 费用报销：项目相关的差旅等费用

#### 业务操作

- **项目盈利分析**：收入 - 成本 = 项目利润
- **预算控制**：设置预算上限，超支预警

## 二、工时表

### 1. 工时记录

#### 填报方式

参见 [[07. 人力资源]] → 工时表

#### 项目特有功能

- **项目选择**：选择归属项目（可选具体任务）
- **计费标记**：标记该工时是否可计费
- **描述**：工作内容描述（客户可见）

---

### 2. 审批与验证

#### 业务操作

- **项目经理审批**：确认工时合理性和可计费性
- **锁定**：审批通过后锁定，不可修改
- **月度汇总**：按月汇总各项目工时

---

### 3. 计费与发票

#### 业务操作

- **生成发票**：根据已审批的工时对客户开票
- **费率设置**：不同角色/技能的不同小时费率
- **预付费抵扣**：从客户预付款中抵扣

---

## 三、计划调度

### 1. 资源管理

#### 字段解释

| 字段 | 说明 |
|------|------|
| **资源** | 人员 / 设备 / 会议室 |
| **可用性** | 正常 / 请假 / 培训 / 不可用 |
| **技能标签** | 具备的技能（如 Python、设计） |
| **费率** | 该资源的小时费率 |

---

### 2. 排班计划

#### 业务操作

- **创建班次**：早班 / 晚班 / 夜班定义
- **自动排班**：基于项目需求 + 资源可用性自动生成
- **冲突检测**：同一资源不可同时安排两个任务
- **员工自助**：员工查看个人排班，申请调班

---

### 3. 利用率分析

#### 报表

- 资源利用率：实际工时 / 可用工时
- 项目资源投入分布
- 未来资源需求预测

## 四、现场服务 (企业版)

### 1. 服务工单

#### 字段解释

| 字段 | 说明 |
|------|------|
| **工单号** | 系统自动生成 |
| **客户** | 服务对象 |
| **服务类型** | 安装 / 维修 / 保养 / 巡检 |
| **优先级** | 普通 / 紧急 |
| **计划日期** | 预计上门服务日期 |
| **工程师** | 派工技师 |
| **状态** | 待派工 → 已派工 → 进行中 → 完成 |

#### 业务流程

```
客户报修 → 创建工单 → 调度派工 → 技师接单 → 现场服务 → 客户签字 → 开票
```

---

### 2. 移动作业

#### 功能

- **手机 APP**：技师通过 Odoo APP 查看工单
- **导航**：一键导航到客户地址
- **工作表**：现场填写服务报告、检查结果
- **拍照**：上传现场照片
- **电子签字**：客户现场签字确认
- **备件使用**：记录更换的备件

---

### 3. 备件管理

#### 业务操作

- **备件需求**：技师在工单上申请所需备件
- **库存检查**：确认库存可用性
- **领用出库**：从仓库领用备件到工单
- **结算**：备件成本 + 人工费 = 服务费

---

## 五、服务台 (企业版)

### 1. 工单受理

#### 字段解释

| 字段 | 说明 |
|------|------|
| **工单号** | 系统生成 |
| **主题** | 问题描述 |
| **请求人** | 提交工单的用户 |
| **类型** | 问题 / 事件 / 变更 / 服务请求 |
| **类别** | 分类（如 IT / 财务 / 行政） |
| **优先级** | 低 / 中 / 高 / 紧急 |
| **状态** | 新建 → 处理中 → 等待客户 → 已解决 → 已关闭 |
| **解决时间** | 实际解决时长 |

#### 受理渠道

| 渠道 | 说明 |
|------|------|
| **邮件** | 发送邮件到支持邮箱自动创建工单 |
| **网站表单** | 门户提交工单 |
| **电话** | 客服接听后手动录入 |
| **实时聊天** | 聊天中一键创建工单 |
| **内部提交** | 员工在系统内提交 |

---

### 2. SLA 管理

#### 字段解释

| 字段 | 说明 |
|------|------|
| **SLA 名称** | 如「金牌服务」「标准服务」 |
| **响应时间** | 首次响应承诺时间 |
| **解决时间** | 问题解决承诺时间 |
| **工作时间** | 计算 SLA 的时段 |
| **升级规则** | 超时后升级给谁 |

#### 业务操作

- **SLA 监控**：工单上显示 SLA 倒计时
- **超时预警**：接近超时前自动提醒
- **超时升级**：自动通知上级经理
- **SLA 报表**：统计各团队的 SLA 达成率

---

### 3. 知识库

#### 业务操作

- **创建文章**：常见问题、解决方案、操作指南
- **关联工单**：处理工单时引用知识库文章
- **搜索**：客服人员快速搜索解决方案
- **客户自助**：将常见问题发布到客户门户

---

### 4. 客户满意度

#### 业务操作

- **满意度调查**：工单关闭后自动发送评价邮件
- **评分维度**：解决速度、态度、专业度
- **满意度报表**：团队/个人满意度排名
- **低分跟进**：对低分评价进行回访改进

## 附录

### 项目管理检查清单

- [ ] 项目范围、目标、里程碑已明确
- [ ] 团队成员已分配，权限已配置
- [ ] 账单方式和费率已确认
- [ ] 任务已分解，截止日期已设定
- [ ] 客户可访问项目门户（如需要）

### 服务台 KPI

| 指标 | 目标 |
|------|------|
| 首次响应时间 | < 1 小时（高优先级） |
| 平均解决时间 | 按类型设定 |
| SLA 达成率 | > 95% |
| 客户满意度 | > 4.5 / 5 |
| 一次解决率 | > 80% |
