# Odoo 权限体系配置指南

在开始配置前，请确保已激活**开发者模式（Debug Mode）**，否则无法看到底层配置菜单。

### 一、 菜单级权限 (Menu Items)

**核心逻辑：** 控制用户进入模块的“入口”。

- **路径：**
  设置 -> 技术 -> 用户界面 -> 菜单项目。
- **配置要点：**

**注意：** 仅隐藏菜单并不代表用户无法访问数据（用户仍可通过搜索或关联字段跳转），这只是第一道视觉屏障。

---

### 二、 模型访问权限 (Access Rights)

**核心逻辑：** 针对整个“表”的 CRUD（增删改查）权限。

- **路径：**
  设置 -> 技术 ->模型。
- **配置维度：**
  | 权限 | 说明 | | :--- | :--- | |
  **读取 (Read)**
  | 是否能看到记录列表和详情。 | |
  **写入 (Write)**
  | 是否能修改现有记录。 | |
  **创建 (Create)**
  | 是否能新建记录。 | |
  **删除 (Unlink)**
  | 是否能物理删除记录（慎用）。 |
- **最佳实践：**
  建议通过**群组（Groups）**来批量管理访问权限，而不是给单个模型硬编码。

---

### 三、 记录规则 (Record Rules)

**核心逻辑：** 垂直权限控制，实现“**同一张表，不同人看不同数据**”。

- **路径：**
  设置 -> 技术 -> 安全性 -> 记录规则。
- **配置关键点：**

- **避坑指南：**
  * 如果一个用户属于多个组，记录规则通常是**并集（OR）**关系，只要满足其中一条规则即可访问。

---

### 四、 字段与按钮级权限 (View Level)

**核心逻辑：** 针对 UI 界面上的具体元素进行权限隔离。

- **配置方式：**
  修改视图（View）的 XML 代码。
- **应用场景：**

**示例：**

- XML

```
<button name="button_validate" groups="account.group_account_manager" ... />
```

- **进阶：**
  如果需要根据
  **单据状态**
  动态隐藏（如：草稿态可见，已确认不可见），应使用
  `attrs`
  属性（Odoo 17 以后使用
  `invisible`
  属性）。
