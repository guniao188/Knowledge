# 项目部

#### 一、 组织架构与基础数据

1. 请简述项目部门的组织架构及岗位职责分工（如：项目经理、现场实施、技术支持、资料员等）。
2. 目前公司主要的项目类型有哪些？（如：EPC总包、设备安装调试、软件开发、纯工程施工等）
3. 项目是否实行项目经理负责制？项目经理在预算控制、人员调配、采购申请上有多大的权限？

#### 二、 核心业务流程

**流程全景**：项目立项 → 计划制定 → 任务执行/资源领用 → 进度汇报 → 变更管理 → 验收交付 → 项目结算

#### 三、 业务流程详细梳理

**1. 项目立项与预算**

- 项目立项的触发条件是什么？（如：销售合同签订后自动立项、中标通知书、内部预立项）
- 立项时是否需要编制项目预算？预算的颗粒度细化到什么程度？（如：按人工、材料、外包、费用大类，还是细化到具体物料编码？）
- 预算制定后，是否允许调整？调整的审批流程是怎样的？

**2. 项目计划与进度管理**

- 目前使用什么工具制定项目计划？（如：Project、Excel、甘特图）
- 项目计划通常需要分解到哪一级？（如：里程碑节点、具体任务、周计划）
- 项目经理如何汇报进度？是按百分比、完成量还是工时填报？
- 当实际进度滞后于计划时，系统或流程上是否有预警机制？

**3. 项目成本与物料管理**

- 项目所需的物料/设备，是根据BOM（物料清单）直接生成需求，还是由项目经理手动申请？
- 是否存在“项目专用料”？如何防止项目专用料被其他项目挪用？
- 项目现场是否设立临时仓库？如有，如何进行入库、出库和盘点管理？
- 项目成本核算目前是按“完工百分比法”还是“完工确认法”？能否实时查看单个项目的预估毛利？

**4. 外包与分包管理**

- 项目执行过程中是否涉及外包或分包？（如：劳务分包、专业工程分包）
- 分包商的进度款支付，是否与项目整体进度或验收节点挂钩？如何管控超付风险？
- 外包产生的费用如何归集到具体项目中？

**5. 项目变更管理**

- 项目变更（如：范围蔓延、设计变更、工期延误）的频率高吗？
- 发生变更时，如何评估对成本和进度的影响？变更签证单如何流转和存档？
- 销售端的合同变更（如增减项）如何同步触发项目端的预算和计划调整？

**6. 项目验收与交付**

- 项目验收的标准和流程是什么？（如：初验、试运行、终验）
- 验收通过后，项目剩余物料（退库）和固定资产（转资）如何处理？
- 项目文档（图纸、验收报告、操作手册）目前如何归档？是否需要ERP系统建立项目知识库？

#### 四、 痛点分析与ERP期望

1. 目前项目管理中，最让您头疼的问题是什么？（如：成本超支无法及时发现、多项目资源冲突、进度汇报滞后、文档版本混乱等）
2. 您希望ERP系统能提供怎样的“项目驾驶舱”或报表？（如：项目进度总览、项目盈亏分析表、人员工时负荷图）
3. 是否有上述调研未覆盖，但您认为对项目管理至关重要的业务内容？如有，请详细描述。
