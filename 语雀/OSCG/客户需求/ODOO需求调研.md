# ODOO需求调研

会议内容：需求调研，不含功能演示

| 姚代娣 | 项目经理兼实施顾问 |
| --- | --- |
| 葛忠彪 | 技术支持 |
| 灵活调配的技术团队 |  |

**一、上线范围**

目前第一个的上线公司抬头有几个？是否涉及多公司及内部公司间交易？

部门架构和职能架构

预计上线时间

换系统的原因或odoo核心功能需求


**二、销售**

销售具体业务流程？

各公司各团队的业务员有多少？

是否有多种价格表？销售员是否可以修改价格表？

有几种币种的销售情况？

销售订单价格是否含税？

销售订单如何开票，收款的流程和方式？

出口报关单据是否需要odoo从设置打印模板设置

是否有退货流程


**三、采购**

采购整体流程是否包含审批

供应商是否固定，是否有多方比价再下单

是否有完整的供应商档案？

退货流程


**四、库存**

备货策略是以销定采还是安全备货、或以采定销

目前仓库和位置有？其存储功能是什么？

是否有专人来管理仓库？

仓库入库是否涉及质检？

现有库存是否精准？如不精准存在什么问题。

仓库出库发货是否涉及审批流程？

产品是支持选配，是否有多个颜色的相同款式


**五、会计**

1. 计价方式是哪种，先进先出还是移动加权平均？

2.目前是否使用财务系统，会计模块目前是如何考虑，先上应收应付，还是全部？

3. 如果涉及多公司，账套如何管理？是否合并账套？内外帐是否一套？
