# 采购业务

流程概要：

采购需求->采购询价->采购下单/催单->预付款请款->采购对账->采购开票->采购付款

### 【采购需求】

1.请购：如研发用料的请购，维修用料的请购。请购->询价/集采->入库一>领用

2.安全库存（MTS、订货点OrderPoint）：自动安全库存，手动安全库存（零安全库存）

3.按需采购（MTO、JIT、零库存策略）

4.主生产计划（MPS）：主要产品/关键产品滚动计划，基于此计划计算采购需求。应用要点：除去MTS和MTO物料，剩下的物料可以考虑MPS计划，如一些贵重的采购周期长的进口物料

5.补货路线：找谁要货，货到哪里去，分为调拨、采购、生产三种。（库存-配置-重新订货原则）

### 【采购询价】

针对非常规采购订单，常规采购已形成供货商

1.询价业务员分配：多个采购业务员，每个业务员负责不同品类的料号询价和采购

2.询价：来自采购需求的询价，来自销售的询价。案例：国外客户新设计了某种纸牌玩具，找国内供应商报价。供应商从市面上分别询价纸张费、纸牌加工费、包装盒加工费，再加上一定利润报价给客户。外贸公司一个业务员一天可能要处理10个客户报价，每20个客户报价可能有一单成交

3.在线询价

4.比价：多个产品，每个产品有不同供应商的不同价格和交期，如何方便筛选出合适供应商和价格

### 【采购下单】

1. 供应商平衡：例如，需求一万件，可能同时找A供应商采购7000件，找B供应商采购3000件。
2. 采购框架合同：在大宗商品采购领域，在一定时期内，按一个锁定的价格，承诺采购一定的数量。这种情况，采购订单都要关联对应的框架合同。
3. 交期跟踪/船期跟踪：预计什么时候到货，国内采购的时候，联系供应商确认交期。进口采购的时候，跟踪供应商交期，供应商装船后，再跟踪船期。
4. 采购交货：分批交货、欠交、多交，让步接收，分别如何处理
5. 供应链金融：例如采购货款100万，自己出20万，找垫资方出80万，货物进入垫资方监管仓库。需要提货的时候，例如计划提货10万，则给垫资方付款8万（加一定利息），垫资方开具提货单，凭提货单到监管仓提货。

### 【采购对账/开票/付款】

1. 采购预付款：
2. 采购月结对账：订单数多少、交货数多少，开票数多少，未开票数多少，本月开票多少
3. 采购开票：对账无误后，供应商按对账数量和金额开发票过来，采购提交发票给财务入账
4. 采购退货：
5. 采购请款及审批：账期到了，采购提交付款申请，财务安排付款。

### 采购交货的各种场景

1、分批交货：指供应商将订单分成几次交付。这种情况通常是因为供应商无法在一个时间点交付所有物品，或者是为了避免库存过多而采取的措施。

2、欠交：指供应商未按照合同约定时间交货。这种情况可能会导致采购方无法按时完成生产或服务，从而对业务造成影响。

3、多交：指供应商超出合同约定的数量交付物品。这种情况可能会使采购方不得不处理多余的物品，增加成本或储存空间的需求。

4、让步接收：指采购方在供应商交付的物品中发现了一些缺陷，但仍然决定接受这些物品。这种情况可能是因为缺陷并不严重，或者是为了避免时间和成本的损失。
