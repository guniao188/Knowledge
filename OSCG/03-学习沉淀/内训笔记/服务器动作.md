# 服务器动作

分为：服务器动作、自动动作、安排的动作

- env
  :触发动作的 Odoo 环境
- model
  :触发动作的 Odoo 模型记录；是一个无效的记录集
- record
  :触发动作的记录；可能无效
- records
  :在多个模式下触发动作的所有记录的记录集；可能无效
- time
  ,
  datetime
  ,
  dateutil
  ,
  timezone
  : 有用的 Python 库
- log(message, level='info')
  :在
  ir.logging
  表中记录调试信息的日志记录功能
- UserError
  : 与
  raise
  一起使用的警告异常
- 要返回动作，请配置:action = {...}

查：

-browse(id)

例如a=model.browse（32）

-search（条件，key-value）

例如b=record.env['purchase.order'].search（[（'name',=',‘Paea31'）,limit=1])

服务器动作2

1. 处理历史遗留问题

1.1 手动发起compute方法(compute方法逻辑变更)

1.2 手动发起onchange方法(导入记录的数据补充)

1.3 手动更新数据，产品的开票策略=>销售明细的开票策略

2. 见不得光的匿名方法

2.1 特定groups(没有外部标识)才能使用

2.2 通过name，id找特定结果集:

if record.categ_id in [1,2,3] and record.name.startswith('哈哈'):

record.write({'categ_id':4})

3. 带动作的界面跳转

record.picking_ids.action_assign()

action = {

'name': 'picking',

4. 跨公司、跨用户的便捷处理

4.1 处理公司间交易数据同步

po = env['purchase.order'].sudo().search([('name','=',record.client_order_ref)],limit=1)

po.write({'remark': record.remark})

4.2 切换用户的操作记录

user1_id, user2_id = 10, 17

record.with_user(user1_id).validate_tier()

record.with_user(user2_id).reject_tier()

4.3 公司属性字段的赋值

for company in env['res.company'].search([]):

account = env['account.account'].with_company(company).search([('code','=',6001)])

record.write({'property_account_receivable_id': account.id})

5. 注意事项

1. 起点不是数据源

for sale_order in records:

for sale_order_line in sale_order.order_line:

sale_order_line.write({'product_id': 3})

2. 不触发onchange事件

sale_order_line.product_id_change()

3. 允许反复触发

# for record in records:

for record in records.filtered(lambda r:r.state == 'draft'):

record.action_confirm()

可以独立使用、每次都即时加载、受环境及记录规则约束，能写5行就能写50行的一个匿名方法
