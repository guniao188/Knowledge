# 长立-外贸订单CRM最终方案与流程

---

## 总体业务主线

```
客户询价
   ↓
【报价单】导入创建 → 自动明细录入（含图片/OE号/客户号/长立号）
   ↓
报价审批（有权限的审批人可在审批时修改）
   ↓
外发报价单（EXCEL）+ 自动生成"报价跟踪"活动（截止=审批+3天）
   ↓
议价（在报价单明细"目标价"中填写）→ 议价审批
   ↓
从【客户】页面选择报价单明细 / 订单明细 / 复制客户货号、数量 → 生成【销售订单】的草稿状态
   ↓
销售订单审批（跟单/计划/采购/生产/仓库 → 侯总终审）→ 转为【合同】
   ↓
发货通知（明细多行选择、按订单数量自动填充发货数）
   ↓
通知计划/采购/计划/生产
   ↓
【发货单】→ 审批流 → 完成
   ↓
提单发放申请（发货完成后按钮，独立审批流）
   ↓
自动生成【发票】（日期=发货日期，可手改）
   ↓
订单界面收款登记 / 提货状态跟踪
```

---

## 一、核心数据模型（最终确定）

### 1.1 新增：报价单表（独立模块）

**原因**：报价阶段的产品可能"无长立编号、快照式存在"，若强制入产品库会污染主数据。以报价单表作为独立快照层。模块字段按照提供的excel和现场会议整理的设计。

**表头关键字段**
客户名称、客户编号、报价单天数、报价单有效期、报价条款、付款条款、销售员（默认为创建人）

**关键字段**（参考客户提供的 Excel）：
- 客户号 / 长立号 / OE号
- 产品名称、规格、材质、参数
- 数量、单价、目标价、小计
- 图片×4
- 备注、议价过程（记录多轮议价原因/结果）
- 最近一次成交价、最近一次订单价（自动带入）

### 1.2 订单转产品处理规则

- 报价明细中优先按**长立编号**匹配产品库。
- 若长立编号不存在：
  - 自动/手动创建新产品；
  - 产品上同步填写：名称、客户编号、OE号。
- 报价单明细的产品字段可脱离主产品库存）。

### 1.3 现有：销售订单 / 合同

- 销售订单 = 报价单转换后生成，**必须来源于报价单**（约束校验，否则不允许保存）。
- 销售订单审批通过后 → 状态转为"合同"。

## 二、编号规则（最终确定）

| 单据    | 编号规则                       |
| ----- | -------------------------- |
| 报价单编号 | 流水号                        |
| 销售订单  | 客户号 + 年 + 3位流水（超过999从01开始） |
| 发货单号  | 销售订单编号 + 2位发货流水            |
| 发票号   | 销售订单编号+ 2位发票流水             |

## 三、整体最终方案

### 3.1 报价管理

| 需求       | 最终方案                                   |
| -------- | -------------------------------------- |
| 报价上传     | 从客户页面增加【上传】按钮，支持 Excel 批量导入；询价附件上传到报价单 |
| 产品图片     | 报价明细、订单明细各支持 **4 张图片**                 |
| 字段扩展     | 参照客户 Excel 定制                          |
| 备注列      | 每行独立【备注】字段                             |
| 快照式产品    | 新增【报价单表】承载快照，允许无长立编号先报价，不强制关联产品表       |
| 历史价格自动匹配 | 增加字段：【最近一次成交价】、【最近一次订单价】（按客户号/长立号自动带入） |

### 3.2 报价审批与销售订单审批

| 需求    | 最终方案                                                     |
| ----- | -------------------------------------------------------- |
| 审批流   | 支持"有权限的人在审批前修改"（销售发起议价前可修改）                              |
| 修改机制  | 审批人可直接改；其他人需通过"重置审批→修改→再提交"                              |
| 报价单导出 | **Excel 导出模板**支持导出含图片，并可以自定义选择字段                         |
| 外发后状态 | 报价单审批通过后自动生成活动待办给销售员：内容名称为【报价跟踪】，截止时间 = 审批日 + 3 天        |
| 中英文模板 | 按客户联系人语言自动切换（先做 PDF，明细合同显示字段：客户号、OE号、4图、参数、数量、单价、小计，中/英） |

### 3.3 议价流程

| 需求    | 最终方案                                                               |
| ----- | ------------------------------------------------------------------ |
| 议价发起  | **不新建报价单**，直接在原报价单明细【目标价】字段填写                                      |
| 修改权限  | 审批过程中审批人可修改；销售在议价前可再次发起修改（触发再次审批）                                  |
| 议价历史  | 明细行【议价过程】字段完整保留：原报价价、议价价、审批意见、确认时间。审批通过后填充到价格列，并在议价过程记录之前价格和最终价的变化 |
| 议价通过后 | 生成最终报价，作为销售订单/合同的直接依据                                              |

### 3.4 合同（订单）管理

**方案**：以 Odoo 原生"订单确认 = 合同"为基础，报价单转销售订单后进入合同状态。

**报价 → 订单四种路径统一实现**：

1. 报价单点【转订单】按钮 → 整单转换；
2. 报价单明细勾选部分产品 → 批量创建订单；
3. **从客户页面**点【生成复购订单】：粘贴/输入 客户号+数量 清单，系统按最新报价匹配价格；未匹配则报错；
4. 客户页面选择多张报价单明细 / 订单明细 → 组合成订单。

**关键约束**：
- 销售订单**必须来源于报价单**，直接创建报错。
- 同一客户+同一客户货号，若不是最新报价，不允许生成订单。
- 订单明细显示"最近报价审批日期"：
  - 超过 **3 个月** → 黄色警示；
  - 超过 **6 个月** → 红色警示；
  - 每日自动任务扫描超期。
  **SKU匹配：**
- 若不存在SKU时，会自动创建产品，并放到类别为【待开发产品库】下
- 长立编号补录：根据长立编号未匹配的产品，创建多个活动待办给工程部。活动名称为：待匹配产品+客户编号+客户产品编号，需要提供到期日期和分配人逻辑。
- 工程部在产品上补维护长立编号，并重新分配产品类别

### 3.5 合同/订单审批（多部门会签）

**审批链**：
```
销售提交
 → 跟单审批
 → 计划审批
 → 采购审批
 → 生产审批
 → 仓库审批
 → 侯总终审
```

- 中间任一节点提出的修改意见→ 由**销售统一执行修改** → 重走流程。
- 通过后销售订单转为【合同】状态。
- 审批流支持自定义修改

### 3.6 跨部门协同（订单明细按钮）

销售订单明细行增加按钮：
- 【通知跟单】
- 【通知计划】
- 【通知采购】
- 【通知生产】
- 仓库通过"发货单"入口查看，无需按钮

（同步显示字段待确认）

### 3.7 合同外发模板

| 项目        | 最终方案                     |
| --------- | ------------------------ |
| 打印字段（英/中） | 客户号、OE号、4张图片、参数、数量、单价、小计 |
| 输出格式      | PDF，在原生的打印上修改订单明细        |

### 3.8 发货管理

| 场景         | 最终方案                                            |
| ---------- | ----------------------------------------------- |
| 发货通知       | 订单明细多行勾选 → 按钮【生成发货单】→ 按订单数量自动填充发货数量             |
| 发货单拆分/合并   | Odoo 原生能力                                       |
| 发货数量校验     | 累计发货 ≤ 合同数量；**最后一次发货**时报错阻止                     |
| 发货单审批流     | 新增：发货单需走审批流才可执行                                 |
| 提单发放申请     | 发货单完成后显示按钮【提单发放申请】，独立审批流；字段：客户、订单号、金额、发货日期、付款条款 |
| 订单列表提货状态字段 | 销售订单列表增加【提货状态】字段，状态为：未提单、部分提单、全部提单              |
| 发货后自动创建发票  | 根据发货数量自动创建发票并确认，发票日期为发货日期                       |

### 3.9 财务联动

- 发货单完成后**自动生成发票**；
- 发票日期字段默认 = 发货日期，可手动修改；
- 收款登记走 Odoo 原生流程。

## 四、审批流全景

| 审批节点          | 触发时机       | 审批人               | 说明                  |
| ------------- | ---------- | ----------------- | ------------------- |
| 1. 报价审批       | 销售提交报价单    | 审批人（有权限可改）        | 通过后生成外发报价单 + 报价跟踪待办 |
| 2. 议价审批       | 销售填写目标价并提交 | 审批人（有权限可改）        | 通过后作为订单生成依据         |
| 3. 销售订单审批（会签） | 报价单转订单后    | 跟单→计划→采购→生产→仓库→侯总 | 通过后订单转为【合同】         |
| 4. 发货单审批      | 发货单创建后     | 待定（默认侯总或指定人）      | 通过后允许出库             |
| 5. 提单发放申请审批   | 发货单完成后申请   | 待定                | 独立审批流               |

## 五、客户页面增强

- 增加字段：报价单有效期、报价单条款【国际贸易术语】，自动带到销售订单上
- 增加【报价单明细】跳转按钮 → 按客户号/长立号过滤该客户历史所有报价明细行。
- 增加【订单明细】跳转按钮 → 同上，展示该客户历史所有订单明细行。
- 增加【生成复购订单】按钮 → 客户号+数量批量匹配最新报价生成订单。
- 增加按钮【导入报价单】，导入自动生成新的报价单
- 增加按钮【生成复购订单】，点击后显示弹窗，按照规则复制进去点击生成报价按钮，如果未匹配的客户编号报错提醒。规则的默认格式为【客户号/数量，客户号/数量...】。取最近的报价的价格进行匹配。
