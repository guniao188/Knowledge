# 凯登功能补充

#### 1.  产品模型扩展与独立信息页开发

- **核心任务**
  ：完成产品模型字段扩展及独立信息展示页面的开发。
- **实施细节**
  ：在产品模型中新增品牌系列（关联配置表）、多维度尺寸（产品/彩盒/内盒/外箱/装箱）、包装规格、上市时间等20余个字段；创建“品牌系列”独立配置表；实现上市时间字段的“年月”格式前端适配；开发独立页面集中展示新增属性。
- **产出价值**
  ：完善了产品主数据体系，实现了产品包装、尺寸、系列等信息的结构化管理，为销售报价、生产物流等环节提供了完整数据支撑。
- **工时投入**
  ：5小时

| 字段名称 | 建议标识 (Key) | 数据类型 | 长度/精度 | 必填 | 说明与备注 |
| --- | --- | --- | --- | --- | --- |
| 品牌系列 | `brand_series_id` | 多对一 (Many2one) | - | 是 | 关联到新建的“品牌系列”配置表 |
| 装箱尺寸-长 | `packing_length` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 装箱尺寸-宽 | `packing_width` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 装箱尺寸-高 | `packing_height` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 箱规 | `carton_spec` | 字符 (Char) | 64 | 否 | 描述包装规格，如“5层瓦楞” |
| 产品副图 | `product_sub_images` | 一对多 (One2many) | - | 否 | 关联图片附件表，支持多图上传 |
| 上市时间 | `market_launch_date` | 日期 (Date) | - | 否 | 前端控件限制为“年-月”格式 |
| 产品尺寸-长 | `product_length` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 产品尺寸-宽 | `product_width` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 产品尺寸-高 | `product_height` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 彩盒尺寸-长 | `color_box_length` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 彩盒尺寸-宽 | `color_box_width` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 彩盒尺寸-高 | `color_box_height` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 端盒尺寸-长 | `inner_box_length` | 浮点数 (Float) | 10,2 | 否 | 单位：cm（注：“抽村”推测为内衬/内盒） |
| 端盒尺寸-宽 | `inner_box_width` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 端盒尺寸-高 | `inner_box_height` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 外箱尺寸-长 | `outer_box_length` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 外箱尺寸-宽 | `outer_box_width` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 外箱尺寸-高 | `outer_box_height` | 浮点数 (Float) | 10,2 | 否 | 单位：cm |
| 立方/箱 | `volume_per_carton` | 浮点数 (Float) | 12,4 | 否 | 单位：m³ 或 cm³ (需确认)，通常指单箱体积 |
| 年龄限 | `age_limit` | 字符 (Char) | 32 | 否 | 如 “3+”, “18+” |
| 产地 | `origin_location` | 字符 (Char) | 128 | 否 | 格式：省+市 |

#### 2.  销售单据格式定制化调整

- **核心任务**
  ：完成报价单、销售订单及销售合同的格式定制化修改。
- **实施细节**
  ：在报价单中集成多个从产品主数据自动带出的新字段，优化界面布局并隐藏非必要信息；保持销售订单界面简洁，未做冗余显示；启动销售合同格式修改，统一对外单据的专业形象。
- **产出价值**
  ：提升了报价单信息的完整性与准确性，优化了销售单据的用户体验，为对外单据的专业化、标准化奠定了基础。
- **工时投入**
  ：5小时（报价单调整）+3-4小时（销售合同修改）

#### 3.审批流程配置与验证

- **核心任务**
  ：完成关键业务模块的审批工作流搭建与逻辑校验。
- **实施细节**
  ：针对销售订单、采购订单、费用报销及付款申请四大核心业务场景，补充并配置了相应的审批节点与流转规则。
- **产出价值**
  ：确保了业务单据流转的合规性与完整性，实现了从单据发起、多级审批到最终归档的全链路闭环管理。
- **工时投入**
  ：4小时

#### 4.委外生产流程贯通与数据完善

- **核心任务**
  ：优化产品委外路线设置，打通生产执行链路。
- **实施细节**
  ：在产品路由中补充了委外加工逻辑，并同步完善了供应商信息价目表，确保系统能准确抓取委外成本。
- **产出价值**
  ：成功跑通了“委外生产”全流程，验证了从委外订单生成到入库结算的可行性，为后续生产业务提供了准确的数据支撑。
- **工时投入**
  ：2小时
